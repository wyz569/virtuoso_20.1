Debugging in Verifier

Virtuoso ADE Verifier (Verifier) provides a comprehensive application to perform requirements-driven verification of analog designs. This appendix helps you with information to debug Verifier when you observe incorrect tool behavior.

Purpose

The appendix focuses on the following areas for debugging the tool:

  • Analyzing the log file.
  • Using the debug mode.
  • Analyzing the location of the implementation run summary data and check when they were created.
  • Debugging performance issues due to frequent check for changes in the implementations.

Audience

This appendix is applicable to all the existing Verifier users having problem with the Verifier functionality or performance.

Overview

As you Verifier user, you might experience some problem or unexpected behavior in the tool. This document helps you to investigate the root cause of such problem areas a bit more. It could lead you to resolve the problems yourself, or you might need to contact Cadence support for further analysis.

Verifier provides many options to debug for incorrect results. This appendix covers the guidelines to be used for debugging the tool. There is an extent to which you can debug the tool else you can contact the Cadence representative.

Use the debugging features of Verifier only while investigating the problem and turn them off afterwards. The features might have a negative impact on the tool's performance and / or might influence the behavior.

Apply these generic guidelines to help Cadence representative find and resolve the problem when you contact Cadence support:

  • Describe the problem as precisely as possible to the Cadence representative.
  • List the steps to reproduce the issue in detail.
  • Check if the issue is testcase specific.
  • Note the ISR stream that has the problem areas. Verify other ISRs as well if the similar issue exists.
  • Collect log files, snapshots, or videos of the issue.
  • Provide a testcase possibly.

This appendix includes following sections:

  • Analyzing the Log File
  • Using Debug Mode
  • Analyzing Implementation Run Summary Data

Analyzing the Log File

The important messages that need immediate attention are displayed in CIW.

Using Debug Mode

The issues related to ADE Assembler might not be visible because they are running in the background when working in ADE Verifier. For debugging such issues, you can run implementations in debug mode in a user interface, where the implementation interface, such as ADE Assembler, gets displayed to monitor the runs. Set the Debug mode, open implementation interface option.

The ADE Assembler UI remains open for you to debug. You can close it manually.  

Analyzing Implementation Run Summary Data

To debug the issues related to an implementation run, check the location of the implementation run summary data using the palswhere env var or the related preference option. Ensure that you are able to access the data at the given location. The data exists in the form of XML files at this location.
