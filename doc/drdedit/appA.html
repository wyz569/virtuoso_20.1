DRD Form Descriptions

This section lists the Virtuoso® Layout Suite Design Rule Driven editing options forms.

Many of the options described in this section have a corresponding environment variable noted next to the description. For more information, see DRD Environment Variables.

DRD Compactor Options

DRD Options

DRD Compactor Options

Use the DRD Compactor Options form to specify how objects in the canvas are compacted and spread by DRD Compactor, the spacing on which changes are based, and the reference objects used to anchor the compaction or spreading. For more information about this feature, see DRD Interactive Compactor.

Compaction Mode lets you specify how compaction and spreading is performed in relation to the specified spacing options. Environment variable: drdEditCompactorMode

Compact and Spread compacts objects that were originally spaced more than the spacing value apart and spreads objects that were originally spaced less than the new spacing value apart.

Compact only compacts objects that were originally spaced more than the new spacing value apart, but does not spread any objects.

Spread only spreads objects that were originally spaced less than the new spacing value apart, but does not perform any compaction.

Spacing Options let you choose the spacing on which compaction and spreading is based. Objects spaced more than the specified value apart are compacted; those spaced less than the specified value apart are spread. Environment variable: drdEditCompactorSpacingMode

DRD Enforce spacing compacts and spreads objects based on the spacing constraints supported by DRD Enforce.

User spacing compacts and spreads based on the value you type in to the field. Environment variable: drdEditCompactorUserSpacing

Apply to Active Layer in Palette applies User spacing to only the shapes on the layer currently defined as active in the Palette. Environment variable: drdEditCompactorActiveLayerOnly

Anchor Mode specifies target objects in the layout against which other objects are compacted or spread. Environment variable: drdEditCompactorAnchorMode

Use last shape in direction of compaction compacts or spreads selected objects by using as the target the last object in the direction of compaction. For example, if you are compacting left, the leftmost object remains static and the remaining objects are compacted (or spread) in relation to that object.

Use first unselected shape in direction of compaction compacts or spreads selected objects by using as the target the first unselected object found in the direction of compaction. If there is no unselected object in the direction of compaction within the visible area of the layout canvas, the selected objects are compacted against the edge of the canvas.

Use DRD Targets compacts or spreads selected objects against a DRD Target object, which can be inside or outside the visible area of the layout canvas.

Maintain relative positions of selected objects maintains the original spacing between selected objects during compaction. For example, if you select three objects for compaction, the object closest to the anchor is compacted against the anchor and the original spacing between the three objects remains unchanged. Environment variable: drdEditCompactorInterOnly

DRD Options

The form comprises the following tabs:

  • Interactive Tab
  • Batch Tab
  • Filters Tab

Interactive Tab

Hierarchy Range sets how many levels of the hierarchy are to be considered during DRD editing.

current cellview the current cellview is considered. This option is selected by default in the drop-down list.

current to bottom the current level of hierarchy to the level specified in the Open to Stop Level (Display Levels Stop) field in the Display Options form is considered.

current to stop level the current level in the hierarchy to the level specified in the Display Levels Stop field in the Display Options form is considered.

current to user level the current level in the hierarchy to the level that you specify. You can specify a value between 0 and 32.

Environment variables: drdEditBatchHierDepth, drdEditHierDepth

Notify provides visual feedback in the form of halos and arrows when design rule violations occur.

Enabled enables the Notify editing mode.

Zoom in Incremental Violation enables a zoomed-in view of the objects involved in violations during incremental violation display.

Smart Snap enables DRD Smart Snap mode, which lets you snap objects to the edges or corners of DRC halos generated during DRD editing. This option is available only when the Notify – Enabled option is enabled. For more information, see DRD Smart Snapping.

Enforce stops pointer movement momentarily when a violation threshold is reached. The halo disappears as soon as the threshold is crossed.

Enabled enables the Enforce editing mode.

Relaxed enables the Relaxed Enforce mode. For more information, see Relaxed Enforce.

Targets Only enables the Targeted Enforce mode. For more information, see Targeted Enforce.

Post Edit generates markers if a layout editing operation results in design rule violations.

Enabled enables the Post Edit editing mode.

Deferred enables the Deferred Post Edit mode. For more information, see Deferred Post Edit.

Marker Limit Specifies the maximum number of post-edit markers to be created per call. The default is 5000.

Environment variable: drdPostEditVioLimit

(ICADVM20.1 Only – 95512) MPT Checks help check the design for conformance to the rules defined in the technology file.

On reports violations for all selected rules, including color-related violations.

Off reports all violations except those related to color.

Only reports only color-related violations.

Conservative reports same-mask spacing violations.

Environment variable: drdEditColorRules

Colorability reports double or triple patterning violations for shapes on a layer.

Environment variable: drdColorability

Color Shorts reports overlaps between shapes with different colors, irrespective of their lock status.

Environment variable: drdColorShorts

Unlocked Shapes reports color-unlocked shapes on a layer.

Environment variable: drdUnlocked

Uncolored Shapes reports uncolored (gray) shapes on layers that support multiple masks.

Environment variable: drdUncoloredShapes

Consider Gray Shapes considers gray shapes as colored and does not report gray shapes as violations while performing color checks on a design.

Environment variable: drdConsiderGrayShapes

Consider Unlocked Shapes As Gray considers unlocked shapes as gray while performing color checks on a design.

Environment variable: drdConsiderUnlockedShapesAsGray

Display helps specify the display elements that should appear when a constraint is violated.

Arrows displays arrows to indicate the optimum spacing between objects.

Environment variable: drdEditDisplayArrows

Edges highlights the edges where the violation occurs.

Environment variable: drdEditDisplayEdges

Rule Text displays a description of the violated constraint in text format.

Environment variable: drdEditDisplayText

Ref Text displays the reference text for the violated constraint from the technology file, followed by the constraint value. When deselected, only the name of the violated constraint is displayed.

Environment variable: drdEditDisplayTechRefs

Halo creates halos around stationary objects when creating or editing an object that violates a spacing constraint.

Environment variable: drdEditDisplayHalo

True Color displays the halo in true color, that is, the color assigned to a layer.

Environment variable: drdEditDisplayTrueColor

Auto Halo Turn Off suppresses halos if the number of violations exceeds the value specified by the user. Only arrows are displayed on the canvas when the user-defined limit is reached.

Environment variables: drdEditAutoTurnOffHalo, drdEditAutoTurnOffHaloLimit

Batch Tab

Hierarchy Range sets how many levels of the hierarchy are to be considered during DRD editing in batch mode.

current cellview the current cellview is considered. This option is selected by default in the drop-down list.

current to bottom the current level of hierarchy to the level specified in the Open to Stop Level (Display Levels Stop) field in the Display Options form is considered.

current to stop level the current level in the hierarchy to the level specified in the Display Levels Stop field in the Display Options form is considered.

current to user level the current level in the hierarchy to the level that you specify.

Environment variable: drdEditBatchHierDepth

Scope helps define the scope of checking in DRD batch mode.

Area specifies the area of check in DRD batch mode.

Environment variable: drdBatchScopeAreaLimit

Current Editable Cellview checks all the shapes in the current editable cellview.

Current View Area checks all the shapes in the current viewable area.

Current Specified Area checks all the shapes in the specified area of the design.

Define Area lets you specify the region in which you want to run DRD in batch mode. Specify the lower-left and upper-right coordinates for the region in this format ((xlower ylower) (xupper yupper)). This option is available only when the Current Specified Area option is selected.

Environment variable: drdBatchScopeSpecifiedArea

Marker Limit Specifies the maximum number of post-edit markers to be created per call in batch mode. The default is 5000.

Environment variable: drdBatchVioLimit

(ICADVM20.1 Only – 95512) MPT Checks help check the design for conformance to the rules defined in the technology file.

On reports violations for all selected rules, including color-related violations.

Off reports all violations except those related to color.

Only reports only color-related violations.

Conservative reports same-mask spacing violations.

Environment variable: drdEditBatchColorRules

Colorability reports double or triple patterning violations for shapes on a layer.

Environment variable: drdBatchColorability

Color Shorts reports overlaps between shapes with different colors, irrespective of their lock status.

Environment variable: drdBatchColorShorts

Unlocked Shapes reports color-unlocked shapes on a layer.

Environment variable: drdBatchUnlocked

Uncolored Shapes reports uncolored (gray) shapes on layers that support multiple masks.

Environment variable: drdBatchUncoloredShapes

Consider Gray Shapes considers gray shapes as colored and does not report gray shapes as violations while performing color checks on a design.

Environment variable: drdBatchConsiderGrayShapes

Consider Unlocked Shapes As Gray considers unlocked shapes as gray while performing color checks on a design.

Environment variable: drdBatchConsiderUnlockedShapesAsGray

Filters Tab

Search Constraints lets you specify strings to filter the constraints for which you want DRD to display violations on the canvas.

Constraint Category lists the constraint categories.

Point to a constraint category to view a list of all the constraints that DRD supports in that category along with the supported modes for the constraints.

Constraint lists the constraints filtered either by using a string or selecting one or more constraint categories.

E (Enforce) indicates whether the filtered constraints are supported in Enforce mode.

N (Notify) indicates whether the filtered constraints are supported in Notify mode.

PE (Post Edit) indicates whether the filtered constraints are supported in Post Edit mode.

B (Batch) indicates whether the filtered constraints are supported in Batch mode.

Ignore Marked Instance excludes the instances from DRD checking that are marked as ignored through the DRD Add Ignore option in the Navigator assistant.

Layer contains the filtering options for layers for which you want DRD to display violations on the canvas.

Palette monitors only those layers for violations that are defined in the Layers panel of the Palette assistant.

Environment variable: drdEditPaletteLayers

Search Layers lets you specify strings to filter the layers.

Layer Category lists the three layer categories, Routing, Device, and Derived.

Layer lists the layers filtered either by using a string or selecting one or more layer categories.

S (Selectability) determines whether the shapes and objects on a selected layer are to be checked by DRD.

The following fields are common to all the tabs:

Configuration file lets you specify the configuration file. The browse button next to it lets you locate and select the existing configuration file and specify the location at which to save the new configuration file.

Save To lets you save the current settings from the DRD Options form to the specified configuration file.

Load From lets you load settings from the specified configuration file into the DRD Options form.
