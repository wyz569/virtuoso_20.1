Batch Checker

Lets you set up the options for performing DRD checks on a layout. For details, refer to Performing DRD Checks.

Field Name Description

Current Editable Cellview

Checks shapes in the entire design for process rule violations.

Area

Checks shapes in the selected area.

Select Area

Lets you select an area within the current cellview.

Marker Limit

Specifies the maximum number of post-edit markers to be created per call. The default is 5000.

Component Overlap Check

Checks if the PR boundaries of two instances placed next to each other overlap.

Layer Stack Editor

Lets you specify the options for the layers. Whenever you add or delete any layer, a confirmation dialog box is displayed. For details, refer to Preparing for Model Creation.

Field Name Description

Thickness

Specifies the default material thickness.

Material

Specifies the name of the material used for the layer.

Conductivity[S/m]

Specifies the electrical conductivity of the material. When the value is 1000 mho/cm or less, the material is considered a dielectric, otherwise it is a conductor.

Permittivity

Specifies the permittivity of the material. It is a measure of capacitance that is encountered when forming an electric field in a particular material.

Loss Tangent

Specifies the loss tangent of the material. Loss tangent refers to the dielectric material's inherent dissipation of electromagnetic energy.

Add Layer

Lets you add the layers. For details, refer to Preparing for Model Creation.

Field Name Description

Layer Name

Specifies the name of the layer being added.

Where

Specifies the position of adding the layer, top or bottom.

Bump and Ball Editor

Lets you specify the physical parameters for IO pads and balls.

Field Name Description

Diameter Top

Specifies the top diameter of the IO pads.

Diameter Max

Specifies the maximum diameter for the IO pads.

Diameter Bottom

Specifies the bottom diameter of the IO pads.

Height

Specifies the height of the IO pads.

Conductivity

Specifies the conductivity for the IO pads.

Export Die

Provides the inputs required for die generation. The inputs are provided using different tabs, Inputs and Outputs, of the form. For details, refer to Exporting the Die.

Field Name Description

Inputs

Die Interface Type

Specifies if there are shape-based IO cells or shapes with overlapping labels in the layout for which the die is being exported.

Front Pin Layer

Use this drop-down list to specify the type of layer that should be read to obtain front side pin information.

Front Label Layer

Use this drop-down list to specify the type of layer where the shapes in the design need to be exported as front side pads in the exported die.

Back Pin Layer

Use this drop-down list to specify the type of layer that should be read to obtain back side pin information.

Back Label Layer

Use this drop-down list to specify the type of layer that should be read to know the back label.

Include unconnected IO Pad cells

Select to include unconnected IO pads while exporting the die.

Pin Numbering

The values specified in this section are used for generating die pin numbers.

Outputs

ECO mode to retain package symbol graphics

If the symbol view for the die that is being exported exists, ensures that the already generated symbols are updated and not recreated.

When this option is selected, pin positions on the existing symbol are preserved. Pin positions are read based on the logical pin name, which is same as the net name in the die. Therefore, if the logical pin name remains same but the physical pin number changes, it is treated as pin update and not delete or add. In this case, the old location of the pin is preserved. However, if pin number is same but the pin name changes, the old pin is deleted and the new one is added in the center of the symbol.

Any pins added to or removed from the die are mapped such that existing symbol pin locations do not change. By default, new pins are added at the center of the symbol. You have to manually move them to the desired location by editing the symbol.

Abstract Library

Specifies the name of the library to be generated after the export die step is successfully completed.

If the Die Export form is launched for a different design, the default Abstract Library/Cell name corresponding to an IC layout name is displayed. However, if it is relaunched from the same design, last saved vales are displayed as the Library/Cell name.

Abstract Cell

Specifies the name of the cell to be generated after the export die step is successfully completed.

The cell name should not be more than 31 characters.

Attach

Enables the fields to attach models during die export.

View name

Displays the view name of the model.

Model file

Specifies the name and the location of the model file being imported.

Import Die Text File

Lets you specify the parameters for importing the die text file.

Field Name Description

Die Text File

Specifies the name of the die text file to be imported in Layout EXL.

Delimiter

Specifies the delimiter options, which function as separators, from the drop-down list box.

Pad Width

Specifies the width of the pad in the unit of measurement that you set for the padstack.

Bind Layout

Provides information about the layout cellview to which the current package die is bound. You can view and edit this information. For details, refer to Update Binding Information.

Field Name Description

Library

Specifies the library to which the current package die is bound.

Cell

Specifies the cell to which the current package die is bound.

View

Specifies the view to which the current package die is bound.

Create TILP

Enables you to create TILPs from the package layout. For details, refer to Creating TILP.

Field Name Description

Base Cell

Specifies the base cellview that has been used to create the TILP.

Target Cell

Specifies the cellview of the newly created TILP.

TILP Type

Specifies the options for the type of TILP to be created.

Die Instance Annotations

Lets you specify the options to display die instance annotations on the canvas. For details, refer to Viewing Die Instance Annotations.

Field Name Description

Annotations

Text Size

Controls the size of text in the die instance annotations.

Bound Die Instance LPP

Specifies the highlight color for bound die instances.

Unbound Die Instance LPP

Specifies the highlight color for unbound die instances.

Display

Displays the die instance annotations on the screen.

Remove

Hides the die instance annotations.

VRF Options Form

Lets you customize the Virtuoso RF-specific editing operations. For details, refer to Setting Virtuoso RF Options.

Field Name Description

IO Movement

Allow abstract IO movement when locked in layout

Specifies whether the IO pins that are locked in the layout die can be moved in the package die.

Restrict IO movement within PR boundary

Controls the area of movement of IO pins in the package die.

Check Locations

Extra IOs

Checks for extra IOs in the package die that are not present in the corresponding layout die.

Missing IOs

Checks for IOs that are not available in the package die, but are present in the layout die.

Mis-aligned IOs

Checks for IOs in the package die that do not follow the same alignment as their corresponding IOs in the layout die.

Aligned IOs

Checks for all aligned pins in the package die.

Connectivity Missing IOs

Checks for IOs in the package die that do not have any valid connections.

Connectivity Mismatch IOs

Checks for IOs in the package die that have connectivity different than their corresponding pins in the layout die.

Fix Locations

Extra IOs

Deletes extra IOs in the package die.

Missing IOs

Creates the missing IOs in the package die.

Mis-aligned IOs

Aligns the mis-aligned IOs in the package die.

Connectivity Mis-matched IOs

Reconnects IOs as per their connectivity in the corresponding layout die.

Create Guides

Use this form to create line-shaped or arc-shaped guides around dies.

Field Name Description

Die Side

Specifies the slide along which guides must be created. Options are: All, North, East, West, and South.

Distance

Specifies the distance from the reference at which the guides must be created.

From edge to the die

Specifies that the Distance must be calculated from the die edge.

Radius

Lets you create arc-shaped guides.

Guide Vertices offset (+/-)

Specifies the distance of the diagonal line drawn from the two vertices of the guide to the center of the die.

Create Bond Wire

Use this form to generate bond wires.

Field Name Description

Create

Specifies what must be created - Bond Wire, Bond Finger, or Both.

Wire Profile

Lists all wire profiles that are defined in the technology file.

Snap To

Specifies whether the components must snap to Bond Finger, Guide, or Die IO.

Uniformly Distribute Objects on Guide

Distributes bond fingers evenly along the guide.

Definition

Specifies a bond finger defintion. To create a new defintion, click Add. To view an existing definition, select it from the drop-down list and click Display.

Alignment

Specifies the bond finger alignment - along wire or orthogonal to die.

Add Bond Finger Definition

Use this form to create bond finger definitions.

Field Name Description

Definition Name

Specifies a unique bond finger definition name.

Shape Type

Specifies the shape of the bond finger - rectangle, oblong, circle, or padStack.

Layer

Specifies the layer in which bond finger must be generated.

width

Specifies the width of the bond finger.

length

Specifies the length of the bond finger.
