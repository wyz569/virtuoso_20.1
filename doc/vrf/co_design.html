Edit-in-Concert

Edit-in-Concert lets you modify a die and package simultaneously. It enables the IC designers and package designers, who are cross-geographically located and work on different platforms, update the designs (die or package) using the abstract files that do not require synchronous or concurrent operation of the tools.

The chapter includes the following sections:

  • Update Binding Information
  • Editing-in-Concert
  • Checking and Fixing IO Pad Locations

Update Binding Information

When a package layout is launched in Layout EXL, its persistent bindings are read and stored in memory. To view these bindings, choose Module – Bind Layout. The Bind Layout form is displayed, which provides information about the Library, Cell, and View of the layout to which the current package die is bound. The default values are the properties set by die export on the corresponding TILP. In order to maintain the binding between the schematic die symbol and the die layout in package, if there is any mismatch in the binding between the schematic and layout instances, you should update the physical binding.

In certain situations, for example, when dies of different technologies are brought together using TILPs, the binding information might be lost during the process. In such situations, use the Bind Layout form to update the binding information.

Viewing Die Instance Annotations

In very complex package designs, it might be difficult to visually distinguish the die instances from other die components. The Module menu includes the Die Instance Annotations command, which lets you highlight the die instances and display annotations that provide basic information about the die instances.

Choose Module – Die Instance Annotations to display the Die Instance Annotations form. Use the options in this form to customize the way in which die instance annotations are displayed.

  1. Drag the Text Size slider to adjust the size of annotation text.
  2. Use the Bound Die Instance LPP and Unbound Die Instance LPP options to assign different highlight colors to bound and unbound die instance LPPs to facilitate easy visual distinction in complex designs.
  3. Click Display to view the annotations on the canvas.
Click Remove to hide the die instance annotations.

The die instance annotations display the following information:

  • Name of the die instance. The instance to which it is bound is displayed within brackets.
  • Mirroring state of the die instance.
  • Flipped state of the die instance.
  • Optical shrink factor of the die instance.
Use the Edit Instance Properties form to change the above values.

Editing-in-Concert

To know more about this feature, watch the Virtuoso RF Solution: Using the Edit-in-Concert Mode to Edit Die Packages and Layouts video available on Cadence Support portal.

The Edit-in-Concert feature lets you view and edit die packages and their corresponding die layouts synchronously. This feature is useful when the die package and the die layout are created independently, for example, when a layout engineer creates a die layout and a package engineer creates the corresponding die package.

  • In die layouts, device placement is driven by factors such as the internal blocks of the design, routing of the layout, and the power requirements of the layout.
  • In die packages, device placement is determined by factors such as the final alignment with the package BGA, the minimum bond wire lengths, and the usage of passive elements in the package.

You can use Edit-in-Concert to view and edit the package and the layout synchronously. Edit-in-Concert lets you:

  • Visualize the IC in the context of the package and see where a change to one die impacts the design of the other.
  • Ensure that the combination of die layout and the die package meets all design requirements, while minimizing the overall cost of production.

In the Edit-in-Concert mode, the dies in a package are superimposed. The current die is displayed in the foreground, available for edit. The dies in the background serve as a template to guide the design.

Launching the Edit-in-Concert Mode

The Edit-in-Concert mode can be launched from the package and board layout. To switch to the Edit-in-Concert mode, with a package die layout open, choose Module – Edit-in-Concert. The package design is displayed on the first tab and the die layouts associated with the die instances in the package are displayed on separate tabs.

In the following example, the given package die includes two TILPs, LNA and PA. In the Edit-in-Concert mode, the package die layout is displayed on the first tab and the die layouts for the two TILPs are displayed on separate tabs.

Here is zoomed-in image of the above design, which shows the package die in the foreground (editable) and the die layout corresponding to the TILP in the background (not editable).

The display levels of the dies in the Edit-in-Concert tabs depends on the stopLevel setting. To open all layout tabs at the same level, set stopLevel to the required display level before launching Edit-in-Concert.

On the die layout tab, the die layout components are displayed in the foreground and the package die components are displayed in the background.

You can also open from a board layout in the Edit-in-Concert mode. Choose Module – Edit-in-Concert with the board layout open.

  • The board design is displayed on the first tab and it contains the package TILP.
  • The second tab contains the package layout corresponding to the TILP in the board. This tab also contains the TILPS corresponding to die layouts.
  • The third tab contains the actual die layout corresponding to the TILP in package layout, which is displayed on the second tab.
You can use the Palette assistant to control the visibility of layers across all dies that are open in the Edit-in-Concert mode. For example, if you hide the visibility of layers A and B in the foreground view, these layers are automatically hidden in the background cellviews. This feature is useful for designs with a large number of shapes in multiple layers, where understanding the layers and shapes in the background cellviews is difficult. Enabling selective visibility of layers in the background cellview is useful in such scenarios.

The Edit-in-Concert mode supports packages with multiple instances of the same die that are bound to the same layout and die footprint. The die selected before launching Edit-in-Concert is considered the active die and is displayed on a separate tab in the Edit-in-Concert mode. The changes made to this die are automatically propagated to other instances of the die. The updates made by the LVA fixer are also propagated to the remaining dies.

The Edit-in-Concert mode does not support die instances that are bound to the same layout but have different die footprints.

Editing in the Edit-in-Concert Mode

Edit-in-Concert supports dynamic modification of location and contents of the dies. Therefore, the dies are always synchronized. Any modification you make to a die is auto-updated on the other tabs in the background. These modifications are immediately visible on all the relevant Edit-in-Concert tabs.

When a bump cell containing bumps has been instantiated multiple times in a layout, moving a bump in the Edit-in-Concert mode results in the movement of the bump at multiple locations. The bump is moved at all locations in which the master bump cell has been instantiated.

Switching from Package view to Layout View

At any point, you can switch from the package view to the corresponding layout view. To do this, right-click the canvas and choose Switch Tabs to Layout from the shortcut menu, as shown below:

To exit the Edit-in-Concert mode, close one or more tabs. A message prompting you to save the changes is displayed. Click Yes to retain the changes. If you click No, the changes made in the Edit-in-Concert mode are lost. As a result, the dies might be asynchronous.

To check for such violations, use the layout versus abstract (LVA) checker. Use the LVA fixer to fix such violations. For more information, see Checking and Fixing IO Pad Locations.

Probing

Probing lets you interactively explore the design and locate the counterpart of a selected object in different windows on different fabrics. In the Edit-in-Concert mode, you can probe nets, bump instances, and Ball Grid Array (BGA) balls. Multifabric net probe lets you probe nets from a package to a die or from a board to a package, and then to a die. When an object, for example, a net, is probed in the package die tab, it is highlighted in the corresponding layout tabs. This section covers the procedures for:

  • Probing a Net
  • Probing Bump or Ball

Probing a Net

To probe a net:

  1. Launch the Edit-in-Concert mode from a package die of the required die instance.
  2. Click either the package tab or the die layout tab.
  3. Right-click the required net in the Navigator assistant.
  4. Choose Probe and select a highlight color from the shortcut menu.

The top-level net geometries (in the foreground) and the net geometries inside the die layout (in the background) are highlighted on the package tab. Similar highlights are seen in the die layout tab.

Checking and Fixing IO Pad Locations

Components of the die layouts and die packages that are not edited in the Edit-in-Concert mode might not be synchronized with each other. Use the Module – Layout Vs Abstract submenu to run the following LVA commands to check for and fix mismatches between a die package and its constituent die layouts:

  • Running the LVA Checker
  • Running the LVA Fixer
The Check IO Pad Locations, Fix IO Pad Locations (and Options) commands for IC fabrics is only available in the Edit-in-Concert mode.

You can run these commands on all or selected die instances or IO pads.

The Check IO Pad Locations, Fix IO Pad Locations, and Launch Edit-in-Concert commands do not run on instances that are bound to a layout with shape-based IO pads. A warning message is displayed listing all the instances that have been ignored.
Alternatively, you can use the Connectivity – Check – Against Source (CAS) command to check the differences between the schematic and the layout. The Check Against Source form includes the Die Layout vs. Die Abstract check box that lets you check for mismatches between a die package and its constituent die layouts. Violations are listed on the CAS tab of the Annotation Browser.

Running the LVA Checker

LVA checker can be run from the board, package, or die layout.

  • If run from board layout, checking is done between package layout and package abstract.
  • If run from package layout, checking is done between die layout and die abstract.

Choose Module – Layout Vs Abstract – Check IO Pad locations from either the package layout or the die layout to start the LVA checker.

Violation markers are created to indicate the following types of violations:

  • Connectivity mismatches
  • Missing IO pads
  • Extra IO pads
  • Misaligned IO pads

Use the Virtuoso RF Options form to select the checks that the LVA checker must run. For more information, see Setting Virtuoso RF Options.

Corresponding environment variable: casDieLayoutVsAbstract

A detailed Die Instance Report is displayed in the CIW. Use the Annotation Browser to view and manage the violation markers.

A detailed Die Instance Report is displayed in the CIW. Use Annotation Browser to view and manage the violation markers.

Running the LVA Fixer

To fix the violation markers created by the LVA checker, select the marker in the Annotation Browser and then choose Module – Layout Vs Abstract – Fix IO Pad locations.

The LVA fixer makes the required changes to fix the violation:

  • Fixes connectivity mismatches. For more information, see IO Pad Connectivity Mismatch Fixes.
  • Fixes the differences in the number of IO pads by doing one of the following:
    • Creating missing IO pads
    • Deleting extra IO pads

    Then, it updates die abstract’s schematic and symbol accordingly. For more information, see IO Pad Number Mismatch Fixes.
  • Aligns IO pads

Use the Virtuoso RF Options form to select the violations that the LVA checker must fix. For more information, see Setting Virtuoso RF Options.

The markers are cleared from the design and the Annotation Browser. The changes made are listed in the Die Instance Report.

IO Pad Connectivity Mismatch Fixes

The LVA fixer addresses the connectivity mismatches that the LVA checker reports for the die abstract, and propagates the changes to the corresponding symbol and schematic views of the die abstract. Therefore, the connectivity in all associated views are synchronized.

Fabric Type of Mismatch Resolution

Non-IC Fabric

The die layout view in the background is the golden view.

IO pads have connectivity in the die layout, but have no connectivity (unconnected IO pads) in the die abstract

The LVA fixer refers to the bump connectivity information in the die layout and establishes similar connectivity in the die abstract. It also propagates the changes to the corresponding die abstract’s symbol and schematic views.

If the IO pad is connected to a net that has multiple IO pads/pins, for example, VDD has three IO pads where the connectivity of any bump is changed to VDD, no update is needed in die abstract’s symbol because the symbol has only one VDD irrespective of the fact that how many IO pads of VDD exist in the die abstract.
If the IO pad is connected to a net that has no existing IO pads in the die abstract, for example, if the IO pad connectivity is changed to pin A that has no existing IO pads, the die abstract’s symbol needs to be updated.

IO pads have connectivity in the die abstract, but have no connectivity (unconnected IO pads) in the die layout

The LVA fixer removes connectivity from the IO pads in the die abstract. It also propagates the changes to the corresponding die abstract’s symbol and schematic views.

IO pads with different connectivities in the die layout and the die abstract

The LVA fixer establishes connectivity in the die abstract as per the die layout and propagates the changes to the corresponding die abstract’s symbol and schematic views.

IC Fabric

The package view in the background is the golden view.

IO pads with different connectivities in the die layout and the die abstract
  • The LVA fixer establishes connectivity in the die layout as per the die abstract.
  • Else, the LVA fixer issues a message indicating that the die layout is not in sync with the die abstract’s schematic or symbol .

IO Pad Number Mismatch Fixes

The LVA fixer updates the die abstract or die layout by adding/deleting the IO pads while fixing in the Edit-in-Concert mode. The following table represents various scenarios for updating the symbol and schematic of the die abstract.

Fabric Type of Mismatch Resolution

Non-IC Fabric

The die layout view in the background is the golden view.

More IO pads on the die abstract compared to the die layout
  • LVA fixer deletes IO pads from the die abstract but it updates the die abstract’s symbol based on the following conditions only:
    • If the IO pad is connected to a net that has multiple IO pads/pins, for example, VDD has three IO pads where one is deleted and two remain, no update is needed in the die abstract’s symbol. The reason being that the symbol has only one VDD irrespective of the number of IO pads of VDD that exist in the die abstract.
    • If the IO pad is connected to a net that has only one IO pad/pin, for example, pin A has one IO pad that is being deleted, the die abstract’s symbol needs to be updated.

Less IO pads on the die abstract compared to the die layout

LVA fixer adds IO pads to the die abstract and updates the die abstract’s schematic and symbol.

IC Fabric

The package view in the background is the golden view.

More IO pads on the die layout compared to the die abstract
  • LVA fixer deletes IO pads from the die layout.
  • The LVA fixer issues a message indicating that the die layout is not in sync with the die layout’s schematic or symbol.

Less IO pads on the die layout compared to the die abstract
  • LVA fixer adds IO pads in the die layout.
  • The LVA fixer issues a message indicating that the die layout is not in sync with the die abstract’s schematic or symbol.

Setting Virtuoso RF Options

Use the Virtuoso RF Options form to customize the Virtuoso RF-specific editing operations. Select Module – Options to display the VRF Options form.

The options in the form let you perform the following tasks:

  • Restrict IO Movement
  • Customize LVA Checks
  • Customize LVA Fixes

Restrict IO Movement

The IO Movement section consists of the following options that let you control the movement of IO pins:

  • Allow abstract IO movement when locked in layout - Controls the movement of IO pins in the package die that have corresponding pins in the layout die. When selected, the IO pins that are locked in the layout die can be moved in the package die. By default, the option is not selected. In this state, IO pins that are locked in the layout die are automatically locked in the package die, and therefore cannot be moved.
  • Restrict IO movement within PR boundary - Controls the movement of IO pins within the die. When selected, the IO pins in a die can be moved only within the PR boundary of the TILP.

Customize LVA Checks

The Check Locations section specifies the checks to be run by the LVA checker. You can choose one or more checks:

  • Extra IOs in the package die that are not present in the corresponding layout die.
  • Missing IOs in the package die for IO pins that are present in the corresponding layout die.
  • Mis-aligned IOs in the package die that do not follow the same alignment as their corresponding IOs in the layout die.
  • Aligned IOs in the package die. This option is not selected by default. This option is useful in certain situations, for example, to create a report about aligned IO pads in the log file, which can be parsed and further used in automated script ware.
  • Connectivity Missing IOs are the IO pins in the package die that do not have any valid connections.
  • Connectivity Mismatch IOs are the IO pins in the package die that have connectivity different than their corresponding pins in the layout die.

To launch the LVA checker, choose Module – Layout Vs Abstract – Check IO pad Locations. Violations are listed on the CAS tab of the Annotations Browser.

For more information, see Running the LVA Checker.

Customize LVA Fixes

The Fix Locations section specifies the fixes that the LVA fixer must do for violations reported by the LVA checker. You can choose one or more fixes:

  • Extra IOs in the package die are deleted. The schematic and symbol of the die abstract are also updated alongside.
  • Missing IOs in the package die are created. The schematic and symbol of the die abstract are also updated alongside.
  • Mis-aligned IOs are aligned as per the alignment in the corresponding layout die.
  • Connectivity Mismatched IOs are reconnected as per the connectivity in the corresponding layout die.

To run the LVA fixer, select either the required die in the layout canvas or the required violations on the CAS tab of the Annotation Browser, and choose Module – Layout Vs Abstract – Fix IO pad Locations. If you do not make the required selections, then the LVA fixer runs on the entire package.
