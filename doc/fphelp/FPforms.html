Analyze Connectivity

Use the Analyze Connectivity form to display information about connectivity within and between blocks. This information is useful during design floorplanning.

Font section specifies the settings of the labels displayed on the flight lines.

Size specifies the font size of the labels. The available options are:

User Defined specifies an absolute font size.

Auto calculates the font size depending on the number of connections.

Color specifies the font color of the labels. The available options are Black & White and Same color as connectivity line.

Transparency specifies the transparency of the labels displayed on the flight lines.

Connectivity section specifies the settings of flight lines.

Width sets the width of the flight lines to either User Defined or Proportionate to connectivity count.

Filter nets hides connections to the selected type of nets—Power, Ground, and Global.

Display Connectivity On Selection Change specifies the mode in which connectivity information is to be displayed when a block is selected. The available options are:

Auto enables dynamic selection of objects and the connectivity information is refreshed automatically.

Manual does nor change connectivity information dynamically.

Enable Net Selection cross-selection of nets between the design canvas and the Navigator assistant.. The available net selection options are:

Exclusive Highlights only those nets that exclusively connect to the selected blocks. Connections to all other blocks are excluded.

Inclusive highlights all nets that are connected to the selected blocks.

Auto-Create Pins

Use the Auto-Create Pins form to search for net shapes in a design and create boundary and buried pins automatically on these net shapes.

Shape Search section provides options to search for net shapes in the design.

Net in specifies the location in which net shapes are to be searched. Select either CellView to search all nets in the current cellview or Selected to search only in the selected nets.

Consider Net Shape on Routing Layers Only restricts the search for net shapes to the routing layers.

Consider Net Shape on Selected Purpose specifies the purpose on which net shapes are to be searched.

Environment Variables: modeOption

Generate Option section specifies generic pin generation options.

Schematic Aware specifies whether the tool must search for corresponding shapes in the schematic view before creating pins.

Delete Pins deletes the existing pins in the layout canvas before generating new ones.

Environment Variables: deletePinCheckbox, deletePinSelection

Auto Created deletes only those pins and associated metal shapes that have been generated by the Auto-Create Pins form.

All deletes all pins in the current cellview.

Check for Duplicate Pins checks for the presence of any similar pins in the design, and if found, does not create a duplicate pin.

Signal Pin Generation section specifies the following signal pin settings.

Boundary Shape Pin specifies that pins are to be created only for the shapes that are touching the PR boundary. Use the following settings to specify the number of signal pins to be generated when multiple boundary shapes belong to the same net:

Single creates a single boundary pin on the highest metal layer.

Multiple lets you specify the layer in which multiple boundary pins are to be created. The available options are Top Layer, which refers to the highest metal layer, and User Layer, where you can specify the layer.

Environment Variable: boundarySquareShapePin

Buried Whole Shape Pin specified that pins are to be created only for the shapes that are either completely inside the PR boundary or are straddling the PR boundary.

Single creates a single buried pin on the highest metal layer.

Multiple lets you specify the layer in which multiple buried pins are to be created. The available options are Top Layer, which refers to the highest metal layer, and User Layer, where you can specify the layer.

Environment Variables: buriedWholeShapePin, buriedMultipleMode

Power Pin Generation section specifies the following power pin settings. Similar to signal shapes, you can choose to create pins for boundary shapes and buried shapes.

Pin Attributes section specifies the following attributes for the new pins to be generated.

Boundary Signal Pin Size lets you specify the size of the auto-created pins.

Auto creates square pins. Only rectangular and square path, pathsegs, and polygonal shapes are recognized for pin creation.

Whole Shape creates pins of the same shape as the corresponding net shapes. For example, square, rectangle, L-shaped, and path pins are supported.

Environment Variable: pinSizeSelection

Pin Layer specifies the layer on which pins are to be generated. The available options are Same as Shape LPP, Same Layer, Custom Purpose, and Custom LPP.

Environment Variables: pinLayers, customPurposeField, customLPPField

Pin Connectivity section lets you select one of the following pin connectivity models for the pins that are generated.

Strongly Connected specifies that the router must connect the pins internally, within the device.

Must join specifies that the router must connect the pins externally, at a higher level.

Must Connect specifies that the router must connect to all pins of the selected terminal.

Environment Variables: pinConnectivityMode

Create Text Label for Pins creates text labels for the pins that are generated.

(ICADVM20.1 Only) Enable Coloring uses the multiple patterning color engine to assign colors to the new pins based on their positions relative to the WSP tracks.

Create Metal Shapes Under Pin creates a metal shape under each auto-created level-1 (soft block) pin.

Path creates a pathSeg metal shape.

Rectangle creates a rectangular metal shape.

Show displays a preview of how the settings would be applied to the design.

Hide closes the preview.

Auto Generate Hierarchy

The Auto-Generate Hierarchy lets you quickly generate a physical hierarchy by applying common parameters. You can define common boundary and pin parameters for the blocks you want to generate. You can specify the shape and size of the boundary that is created when the soft block is generated.

Stop View specifies the required physical view names to be used to determine the corresponding physical views.

Switch View specifies the names to be used to create a physical view for blocks if the stop views are absent in the hierarchy.

Create View specifies the name to be used for the new layout view created when no matching stop views are found. Environment Variables: autoGenHierCreateView

physconfig View specifies the name of the physical configuration view that provides information about the schematic-layout pairs. Environment Variables: autoGenHierPhysConfig

Boundary Attributes section provides options to define the boundary.

Use Area Estimator calculates the boundary based on the following area estimator functions:

PR Boundary Based sums up the polygonal area of the individual instances to derive the overall area estimation.

BBox Based sums up the minimum bounding box area of the individual instances to derive the overall area estimation.

Add lets you register custom area estimation functions and use them.

Aspect Ratio specifies the width-to-length ratio of the boundary. The default value is 1, which specifies a square boundary.

Area Utilization specifies the percentage of area within the boundary that must be filled. The default is 25%. Environment Variables: autoGenHierDefaultArea

Pin Attributes section specifies the following common settings for all the soft block pins:

Layer Purpose Pair specifies the LPP on which the pins must be generated.

Height specifies the pin height.

Width specifies the pin width.

Create Labels specifies whether labels are to be generated along with the soft block pins.

Initialize Instances Outside Boundary generates all the instances and pins outside the PR boundary, without any overlap. The placement statuses of the instances are set to none.

(ICADVM20.1 EXL Only) Hierarchy Controls section defines settings for hierarchical instances.

Generate Hierarchy to Level defines the level up to which the hierarchy must be generated. The default (maximum) value is 31. Environment Variables: autoGenHierDepth

Last Level Generation specifies the state in which the last level of hierarchy is to be generated. The available options are:

Black Box generates the last hierarchical level as a black box containing only the PR boundary and pins.

Flatten generates the PR boundary and pins in the last level. If any additional stop views are found inside the hierarchy, those levels are flattened and displayed.

Show Unplaced Instances on Descend specifies whether unplaced instances are to be displayed when you descend into the hierarchy. The available options are:

Yes (default) shows all unplaced instances.

No hides all unplaced instances.

Select displays a popup message each time you descend into a hierarchy, which lets you specify whether unplaced instances are to be displayed or hidden.

Environment Variable: showUnplacedInstances

(ICADVM20.1 EXL Only) Place Instances and Pins section specifies the following pin placement settings:

Place Instances like Schematic places the generated soft block instances inside the PR boundary as in the corresponding schematic view.

Run Pin Optimization runs Pin Optimizer hierarchically on the generated blocks. You can select one of the following modes:

Top Down runs Pin Optimizer to derive soft block pin positions hierarchically from level-0 until the leaf level.

Bottom Up follows the same behavior as described in Top Down, but runs in the opposite direction.

Environment Variables: autoGenHierRunPinOpt, autoGenHierPinOptMode

Environment Variable: blockAnnotationTextLPP

mm^2: Square millimeters

du^2: Square database unit

Environment variables: blockAnnotationArea, blockAnnotationAreaUnits

Shrink Factor specifies the factor by which each edge has to be shrunk. Environment Variable: blockAnnotationShrinkFactor

Rotation specifies the block orientation information. Environment variable: blockAnnotationRotation

Display Options section lets you customize the following text settings:

Text Size sets the size of block annotations to one of the following values:

Auto calculates the annotation size such that all annotations are inside the instance boundary.

Custom provides a slider to drag to the required value.

Environment variable: blockAnnotationTextSize

Block Placer

Use the Block Placer form to automatically place all hard and soft blocks in a design and minimize the wire length and chip area.

General tab lets you define the chip dimensions, cost order, and other options.

Chip Dimensions section has the following options:

Fixed Die specifies that the block placer cannot alter the chip dimensions. This option is selected by default if the Boundary Area constraint is unavailable in the design.

Minimum I/O to Core Distance specifies the distance between the IOBox and the coreBox (the spacing between the pads and the blocks).

Minimum Boundary to Instance Distance specifies the minimum distance between the design boundary and instance edge. This option is available only when there are no IO pads in the design.

Specify Chip Dimension Range specifies the chip size with respect to one of the following:

Width and Height activates the PRBoundary Width Range and PRBoundary Width Range options.

Area and Aspect Ratio activates the PRBoundary Area Range and PRBoundary Aspect Ratio (W/H) Range options.

Cost Order section specifies the order that the block placer has to follow to minimize cost. The cost with the highest priority is minimized first.

Default Cost Order restores the original cost order.

Options section specifies the following placement options:

Default Minimum Distance between Instances specifies the default minimum distance between instances.

Optimize Top-Level Pins specifies whether the block placer has to optimize the top-level pins after block placement.

Effort specifies the effort that block placer should make before arriving at the possible solutions.

Enable Multi Threading reduces the time taken by the block placer to place a large number of macros.

Enable Congestion Based Refinement is enabled only if congestion data is available. When this field is enabled, the congestion value is taken into account during block placement.

Number of Iterations specifies the maximum number of iterations that can be performed between the congestion-based refinement block placer and the Global Router to bring congestion within the limits.

Constraints tab specifies the constraints that the block placer has to use to converge on a solution.

Soft Block Constraints tab specifies the side constraints for soft blocks.

Soft Block Size Constraints table lists all the soft blocks in the design cellview.

Area specifies the area of the selected soft block.

Minimum AR and Maximum AR specify the minimum and maximum aspect ratio of the selected soft block.

Update All applies changes to all soft blocks.

Update Selected applies changes to only the selected soft blocks.

Reload Constraints from Constraint Manager reloads constraint values from the Constraint Manager.

Instance Constraints tab specifies the instance constraints for soft blocks.

Instance To Instance Edge Distance section specifies the distance constraints between any two instances.

Instance to Instance Distance table is populated from the Instance to Instance Distance constraints present in the Constraint Manager.

Instance1 and Instance2 list all the instances in the design. You can select the instances on which the minimum and maximum distance constraint are to be applied.

Minimum and Maximum specify the minimum and maximum edge-to-edge distance values.

Instance To Boundary Distance Constraints section specifies the distance constraints between an instance and the design boundary.

Instance to Boundary Distance table lists all the Instance to Boundary Distance constraints present in the Constraint Manager.

Instance Name lists all instance names.

Minimum Distance and Maximum Distance specify the minimum and maximum instance-to-boundary distance values.

Boundary Edge specifies the edge to which the constraint is to be applied.

Reload Constraints from Constraint Manager reloads constraint values from the Constraint Manager.

Relative Constraints section specifies the order and alignment constraints to be applied.

Order/Alignment table specifies the alignment constraints to be applied.

Reference Instance specifies the reference instance for calculating the order and alignment.

Alignment Side specifies the edge for which the constraint is to be applied.

Relative Position specifies the relative position of the target instance with respect to the reference instance.

All Instances lists all the instances in the design except the fixed and locked instances. You can select multiple instances in this field and move them to Target Instances by using the up arrow button.

Add creates a new constraint.

Update modifies the selected constraint.

Delete deletes the selected constraint.

Create Pin From Label

Use the Create Pin From Label form to create pins from labels and text displays in the layout cellview.

Scope section defines the scope for seracing labels

Label in specifies the scope for selecting labels. The available options are:

Cellview considers all top-level labels in the current cellview as target labels.

Selected considers the selected labels in the current cellview as target labels.

LPP Filter restricts the search only to the specified layer-purpose pairs. All other LPPs are ignored.

Consider Instance Labels includes instance labels in the search results.

Shape Search section defines options for searching for labels.

Search Depth specifies the level up to which shapes must be searched. For example, if Search Depth is set to 4, then levels 0 through 4 are searched for the required shapes. The default value is 0, which refers to the top-level (level 0).

Select Shape specifies the shape that must be used for pin creation - First or Largest. First indicates that the first shape that matches the specified criteria must be used. The search is stopped as soon as a suitable shape is found. When set to Largest, all matching shapes are retrieved; then the shape with the largest area is identified and used.

Use Shape On specifies the layer property based on which the search results must be filtered out—Metal or Poly Layer or Same Label Layer.

Apply Search Depth for Instance Labels uses the specified Search Depth for instance labels.

Create Pin if Connectivity Exists creates pins only for the labels that are associated with existing nets, terms, or pins in the current cellview. All other labels are ignored.

Pin Attributes section defines settings for the pins to be generated.

Pin Size specifies the size and shape of the pins to be generated. The available options are:

Auto creates square pins inside the shape. The smaller dimension (width, height) is chosen as the pin side of the square pins. Pins are created along the edge of the PR boundary, so that they are accessible for defining connectivity.

Whole Shape creates pins of the same shape as the labels.

Custom (X and Y values) lets you specify the size of the pins by specifying the width and height values. The same size is applied to all output pins.

Pin Layer specifies the layer on which pins are to be generated. The available options are:

Same as Shape LPP creates pins on the same layer as the overlapping shape.

Same Layer Custom Purpose uses the same layer as the overlapping shape, but lets you specify the purpose.

Custom LPP lets you specify both the layer and purpose on which pins must be created.

Attach Label to the Pin attach the labels to their corresponding pins.

Virtual Connect

Create Whole Shape Pin for Labels in the Power section creates power pins of the same shape as their corresponding labels. Click Define to display a list of labels. Select the labels for which power pins must be created.

Show in the Preview section displays a preview of the pins that would be created.

Hide closes the preview.

Create Pin From Net Shapes

Use the Create Pin From Net Shapes form to create pins from existing net shapes in the layout cellview. Only top-level pins can be created using this method.

Net in specifies whether net shapes are to be searched in all or selected nets in the current cellview. The available options are CellView and Selected.

Consider Net Shape on Routing Layers Only restricts the search to the net shapes that are located on routing layers.

Create Pin on Top Layer Only creates pins only on the top layer shape.

Create Pin on Connected Shapes restricts the search to those net shapes that are connected to top-level nets.

Consider Level-1 Connected Shapes includes net shapes that are connected to level-1 nets in the search. This option is available only when the design has level-1 shapes.

Check for Duplicate Pins checks for existing pins with the same names to ensure that duplicates are not created.

Schematic Aware ensures that pins are created only for those nets that have corresponding terminals in the schematic view. Pins are not created for internal nets.

Create Label creates labels over pins.

Options displays the Set Pin Label Text Style form, which can be used to define pin label settings.

Pin Layer section specifies the LPP on which the pins must be created. The available options are:

Same as Shape LPP creates pins on the layers on which the net shapes are located.

Same Layer, Custom Purpose uses the same layer as the net shape, but lets you specify the purpose.Custom LPP specifies the layer and purpose on which pins must be created.

Show in the Preview section displays a visual depiction of how the pins would be created; actual pins are not created yet.

Hide closes the preview.

Electrically Aware Pin Sizing

(ICADVM20.1 EXL Only) Use the Electrically Aware Pin Sizing form to identify pins that do not meet electrical requirements and automatically fix the violations.

Run On specifies the pins on which the command is to be run.

All runs the command on all top-level and level-1 pins

Selected runs the command only on the selected pins.

Find and View under-sized pins identifies violating pins.

Select Pin selects all the detected under-sized pins. You can perform further actions on the selected pins, for example, you can move or resize them.

Highlight Pin highlights the under-sized pins for better visibility.

Highlight Violating Edge highlights only the violating edge of under-sized pins.

Show edge dimension (actual vs desired) displays the current edge dimension and the desired electrically-aware dimension of the violating edge.

Resize under-sized pins fixes the pin violations.

Violating edge only resizes only the violating pin edges to the desired electrically-aware width.

Retain pin selection retains the selection of pins after they are resized.

Retain highlight on original pin retains the highlights on the pins that were selected for resize.

Maintain abutment to PR-Boundary ensures that pins that are initially abutted to the PR boundary retain the abutment even after they are resized. Pins that are not initially abutted remain at the same position by maintaining the center of the pins at the same location.

Show in the Preview section displays a preview of the pins after they are resized before updating the layout.

Hide closes the preview.

Macro Coloring section provides options to assign different highlight colors for hard and soft blocks so that they are visually distinguishable in the layout canvas.

Soft block color specifies the highlight colors for soft blocks. Environment variable: softBlockColor

Hard block color specifies the highlight colors for hard blocks. Environment variable: hardBlockColor

Create Instances specifies that all instances are to be re-generated. Environment variable: initCreateInstances

Create Pins specifies that all pins are to be re-generated. Environment variable: initCreatePins

Pin Placement section defines the following settings:

Respect layer priority for valid routing layers constraint on net specifies that the layer priority must be followed while placing pins during pin optimization.

Default blockType (when unknown) specifies the default block type for soft blocks when the assigned block type is not custom or digital. The available options are:

Custom places pins and PR boundary on the manufacturing grid.

Digital places pins on the routing grid and the PR boundary on the placement grid.

As-Is (XY Snap Spacing) specifies that the pins and PR boundary must placed on the grid defined by the X Snap Spacing and Y Snap Spacing options in the Grid Controls section of the Display Options form.

Environment variable: defaultBlockTypeWhenUnknown

(ICADVM20.1 Only) Enable Deep Hierarchy Editing of Soft Blocks enables viewing and selection of soft-block PR boundary, instances, and pins from any hierarchical level.

I/O Pad Placer

Use the IO PAD Placement form to place PADS on IO rows.

Style section lists the supported styles.

Connectivity Based places PADs in the rows minimizing the net length between the PADs and the placed blocks in the design.

Non Connectivity Based distributes the PADs around the periphery of the design boundary without considering the connectivity between PADs and blocks placed in the design.

Spacing section defines the spacing between the PADs in a given row.

LEFT, RIGHT, TOP, and BOTTOM refer to the row sides.

None specifies that all the placement constraints are to be turned OFF by default and pads placed with spacing value 0.

Even distributes PADs evenly (with same spacing between PADs) in the designated row.

Abut guides the placement engine to abut the PADs in the designated row. The selection of abutment will depend upon the row side.

Custom specifies a custom spacing value. The placement engine honors this value while placing PADs.

Merge and Must Connect Pins

You can use the Merge and Must Connect Pins form to merge two or more overlapping rectangular, rectilinear, or path pin shapes into a single rectilinear shape.

Mode specifies the pins to be selected for merge.

All applies to all pin shapes.

Selected applies to the selected pin shapes.

Related Environment Variable: mergePinsSelMode

Pin Connectivity specifies the connectivity settings for the merged pin.

Must Connect Pins Without Creating New Nets (default) specifies that the connectivity of the pin shapes is to be retained. The must connect all pins setting is applied to the existing terminals.

Must Connect Pins Using New Nets specifies that new nets, terminals, and pins are to be created for each pin shape island. The new terminals are connected and are placed together in a must connect group.

Strongly Connect Pins specifies that a single connection point on a highly conductive metal is to be used to provide the required connectivity.

Related Environment Variable: mergePinsConnMode

Suffix String specifies the suffix to be added to the new nets, terminals, and pins that have been created on the merged pin islands. The default value is _MJ, which is applied when Must connect pins using new nets is selected.

Related Environment Variable: mergePinsSuffix

Use the Pin Checker form to run various checks on pins and report the results.

Check Two Pin Nets section lists checks that are applicable only to the nets that are connected to two pins:

Mis-aligned checks whether the connected pins are correctly aligned.

Layer mismatch checks whether the connected pins are on the same layer.

Size mismatch checks whether the dimensions of the connected pins are the same.

Abut checks whether the connected pins are abutted to each other.

Overlap checks whether the connected pins overlap each other.

Check All Nets section lists checks for all nets. Violations are reported in the Pin Check Report displayed in the CIW.

Multi pin nets identifies nets that have multiple pins and lists their names in the Pin Check Report.

Pin Threshold restricts the check to nets that have the specified number of pins.

Pin layer direction mismatch with side identifies the pins for which pin layers do not have tracks in the specified pin side. This option is applicable only to pins that have side layer constraints.

Pin Check Report section lets you customize the information displayed in the Pin Check Report.

Summary generates a summary report about the checks performed by the Pin Checker.

Detailed generates a detailed report with information about nets as per the above settings.

Pin Group Guide

Use the Pin Group Guide form to create pin groups and guides interactively.

Group Name specifies the name of the new pin group guide.

New generates a unique group name.

Delete deletes the selected pin group guide.

Refresh loads default values from the database, technology file, and constraint.

Preserve Order specifies that the order of pins in the pin group constraint are to be honored and pins are to be optimized in that sequence.

Foreign Pins Restrictions section includes the following options:

Exclusive does not honor foreign pins present within the pin group guide area.

Group Constraints section includes the following options:

Layer Priority specifies that pins are to be optimized in the order in which they are listed in Valid Layers.

Pin-Pin Spacing specifies the spacing between adjacent pins on each layer.

Keepout Spacing defines the area around the pin within which no object can be placed.

Pin Width specifies the width of pins for each layer.

Pins tab provides the following options:

Pin Placement

Use the Pin Placement form to place and optimize pins your design.

Pin Planner tab specifies how pins are to be placed in the design.

Edit lets you specify the scope for running the Pin Planner.

Top Level Pins applies the form settings only to the top-level pins in the design.

Auto-Switch to Level-1 automatically zooms to the selected soft block and switches to Level-1 Pins mode when you click a soft block in the design canvas. This option is off by default, which means that when you click a soft block, the soft block is selected, but you remain at the top level.

Level-1 Pins applies the form settings to pins of the selected soft block in the design. Select the required soft block from the drop-down list.

If you have already selected a block in the canvas, that block is automatically selected in the Pin Planner tab. If you have more than one block selected in the canvas, the first block you selected is the one selected in the form.

Filter filters the pins listed in the form. Use a combination of the two pull-downs to filter the pins that are displayed. Alternatively, you can use the By Name field to filter the pins by name.

Edge filters pins based on the edge to which they are assigned. The available edge options are All, Left, Top, Right, Bottom, Level-1 pin, Top-Level Route, To Hard Block Multi Pin, and Hard Block Multi Pin.

Status or Type filters pins based on their placement status or pin types. The available options are All, Placed, Unplaced, Locked, Fixed, Ordered, Unordered, I, O, and IO.

By Name filters pins based on their names. The default regular expression for this filter is “.*” (dot asterisk), which implies that a name filter is not applied. Therefore, all the pins in the design are displayed. Other supported regular expressions are:

“ ” (Blank): Is equivalent to the default regular expression and it implies that the name filter is not being applied. Displays all the pins.

“in.*”: Displays all the pins that have names starting with “in”.

“.*in.*”: Displays all the pins that contain “in” in their name.

“.*in”: Displays all the pins that have names ending in “in”.

    • The “in” in the valid regular expressions above can be replaced with any other expression, such as “VDD”.
    • The filter string “*.*” is not a valid regular expression. Use a valid regular expression instead.
    • The By Name field is case-sensitive.

Pin table lists all the shape pins found in the current scope. It does not list pins that are vias. The entries are expressed as termName:pinName:figName because the pin figure is the physical entity that you are placing. If a pin has more than two pin figures attached to it, but the pin figures have the same name, they are represented by a single entry in the pin table.

The other columns in the pin table are the Edge to which the pin is assigned, the Order in which the pins are placed, and the Status, Type, Layer, Width, Height, and Connectivity (ICADVM20.1 Only) of the pins. You can sort the table by any column by clicking the relevant column header. Click again to reverse the sort order.

Message provides a hint about your next action.

Attributes applies the following pin attributes to the pins in the pin table.

Edge specifies the edge on which the selected pins are to be placed. The options are Left, Top, Right, Bottom, Level-1 pin, Top-Level Route, To Buried Pin (ICADVM20.1 Only), Level-1 Route, To Hard Block Multi Pin, Any, and As is. The As is setting allows the edge settings of pins to remain intact when changing the Order or Placement Status Constraint properties.

Depending on the block type, the pin edges are snapped to either the manufacturing grid or the routing grid. If the block type is custom, the pin edges are snapped to the manufacturing grid. If the block type is digital, the pin edges are snapped to the routing grid.

Order specifies the order in which the selected pins are to be placed. You can specify an order only if you have assigned the pins to an edge.

To place the pins unordered, leave the field blank or type “-”. To specify an order for the pins, enter a value.

Placement Status Constraint specifies the status to be assigned to the pins after they are placed.

Lock at Placed Location places pins at an appropriate locations on the boundary and then locks them at that location. A locked pin cannot be moved either by the automatic placement functions or manually by the user.

Fix at Placed Location places pins at the appropriate location on the boundary and fixes them at that location. A fixed pin cannot be moved by the automatic placement functions, but can be moved manually, for example, using the Edit – Move command.

No Constraint (Floating) specifies that the selected pins will neither be locked nor fixed after they are placed. A no constraint, floating pin can be moved freely by the automatic placement functions and by using manual editing functions.

As is does not change the status of the selected pins after placement. Use this option to preserve differing status settings on the selected pins.

Shape specifies the following pin shape attributes:

Layer specifies the layer on which the pin shapes are to be drawn. Layer purpose names are filtered dynamically as you type a value in the Layer field.

By default, the width, height, and layer of the pin selected in the pin table is displayed. When you change the layer, Pin Planner checks whether the pin width and height values are valid. These values cannot be lesser than the layer minwidth. Therefore, when you select a different layer, if the pin width and height values retrieved are equal to or greater than the layer minwidth, the values are displayed in the form. If the pin width and height values are less than the layer minwidth, the layer minwidth value is displayed in the corresponding field.

Width specifies the width of the pin shape to be generated.

Height specifies the height of the pin shape to be generated.

Re-Place Pins specifies which pins are to be updated with the new attributes when you click Apply.

All Edges specifies that pins on all edges can be moved by the placer.

Selected Edge specified that only the pins assigned to the selected edge can be moved. All other pins are fixed in their current position.

Selected Pins means that only the selected pins can be moved by the placer when you click Apply. All other pins are fixed in their current position.

Apply applies the specified pin attributes to the selected pins.

Pin Spacing specifies how the selected pins are to be spaced. Select the required pins, set the spacing attributes, and click one of the spacing buttons depending on which of the selected pins is to be assigned as the reference pin. Pins are spaced only by clicking a spacing button in this section and not by clicking Apply in the Attributes section.

Value specifies the distance by which the selected pins are to be spaced.

Spacing type specifies how the spacing value is to be applied.

center-center applies the spacing value to the center lines of each of the pin shapes to be spaced.

edge-edge applies the spacing value to the edges of the each of the pin shapes to be spaced.

Update Constraints automatically updates the constraints that are applicable to the design (for example, alignment constraints on the selected pins) based on the settings in the Pin Spacing section. If a pin to be spaced has the alignment constraint set, the placer reads the constraint to determine the appropriate access direction for the pin.

Space from LEFT/BOTTOM pin uses as its reference pin the leftmost pin on the top and bottom edges (0 in the example below) and the bottom pin on the side edges (C).

Space from CENTER pin uses the center pins on all edges as reference. On the left edge in the example above, this is pin B. On the top edge, the pin spacer divides the total number of pins by 2 and uses the result as the reference pin (2).

Space from RIGHT/TOP pin uses the rightmost pin on the top and bottom edges (3) and the top pin on the side edges (A).

Place As In provides the following options:

Schematic places pins in the same relative positions as in the schematic view.

Symbol places pins in the same relative positions as in the schematic view.

Iterated Pins specifies the way iterated pins are to be listed.

Interleave interleaves the bits of the selected iterated pins by applying an order constraint that specifies how the individual bits are to be placed.

Uninterleave reverses the interleaving of the selected iterated pin bits.

Expand shows all the individual bits of iterated pins.

Collapse shows bus notations for iterated pins.

All performs the Expand or Collapse function on all the pins in the list box.

Selected performs the Expand or Collapse function only on the selected pins.

Change Order lets you change the order of pins in the pin table.

Arrow left/down moves the selected pins down in the ordered list.

Arrow right/up moves the selected pins up in the ordered list.

Swap order inverts the order of two selected pins in the list.

Create lets you create horizontal and vertical rail pins.

VRail elongates the selected pins vertically from the top edge of the boundary to the bottom.

HRail elongates the selected pins horizontally from the left edge of the boundary to the right.

Zoom section lets you zoom the display in the canvas to the currently-selected pins.

Zoom ON enables the zoom functionality. This makes the slider active. You can then use the slider to set the desired zoom level. The acceptable zoom range is between 0 and 4. Level 0 provides the minimum zoom and level 4 provides the maximum zoom.

Alternatively, you can use the environment variable, vpaAutoZoomEnable, to enable the auto zoom functionality. This ensures that the zoom functionality is automatically enabled when you open the layout view. To set the desired zoom level, you can use the environment variable, vpaAutoZoomLevel.

If you set the auto zoom level to a value less than 0 when using the vpaAutoZoomLevel environment variable, the minimum zoom level will automatically default to 0. Likewise, if you set a maximum value higher than 4 using the environment variable, the maximum zoom value will automatically default to 4. The zoom range lies between 0 to 4 irrespective of the method you use to set the zoom level.

Pin Optimization tab lets you to position pins to obtain the shortest possible net length.

Mode specifies the pins to be optimized.

All optimizes all pins in the design.

Selected optimized only the selected pins. In this mode, you can select High Effort to get better results.

Environment variable: pinOptMode, selectedOnly

Pin Spacing section specifies the minimum spacing between the pins.

Use Table Spacing scales up the spacing values as per the table spacing rules defined for different widths of the pins.

Auto indicates that the minimum spacing values are to be based on the line-line spacing mode.

Custom enables the Options and Advanced Options buttons. Click Options to display the Pin Spacing Options form, which lets you specify the pin spacing values for each layer. Click Advanced Options to display the Pin Spacing Advanced Options form.

Environment variable: customPinGrid

Pin Offset specifies the distance between the adjacent side and the center point of the first pin.

Auto places the first pin on a grid such that it is aligned to the corner pins of the adjacent side and that there is no spacing violation.

Custom enables the Options button. Click this button to display the Pin Offset Options form, which provides options to specify the pin offset values for all layers in the design. For more information, see Pin Offset.

Environment variables: offSetByGrids, offSetBySpacing, offSetMode

Target Pin Side specifies the Side of the block or the top-level PR boundary on which pins are to be placed. The valid values are: Left, Right, Top, and Bottom. For more information, see Target Pin Side. Environment variable: targetPinSide

Pin layer/side constraints provides options to switch the pin layer depending on the side of the block on which the pin is placed by Pin Optimizer. For more information, see Pin Layer/Side Constraints. Environment variables: enablePinLayerSideConstraints, bottomSidePinLayersConstraints, leftSidePinLayersConstraints, rightSidePinLayersConstraints, topSidePinLayersConstraints

Match Fixed Pins Layer/Size provides options to change the layer and purpose of the 'to be placed' pins to match the layer and purpose of the connected fixed pins. For more information, see Match Fixed Pins Layer/Size.

Place Rectangular Pin is applicable only to rectangular pins. It specifies the pin edge to be abutted to the PR boundary. The valid values are: As Is, Short Edge, and Long Edge. For more information, see Place Rectangular Pin. Environment variable: rectangularPinEdgeOnPRB

(ICADVM20.1 Only) Honor Current Pin Location snaps pins (selected or all) to their nearest WSP tracks by moving them by a minimum distance from their current position. For more information, see (ICADVM20.1 Only) Snapping Pins to WSP Tracks. Environment variable: honorCurrentPinLocation

(ICADVM20.1 Only) Congestion Aware runs Pin Optimizer in the congestion-aware mode. For more information, see Congestion-Aware Pin Placement (ICADVM20.1 EXL Only). Environment variable: enableCongestion
