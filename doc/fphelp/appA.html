Area Estimation Framework

The area estimation framework enables you to register your own area estimators or use previously registered area estimators.

This chapter includes the following topics:

  • Register Area Estimators
  • Other Utility Functions
  • Sample Area Estimation Procedures

You can either directly type the area estimator functions in the CIW or define them in the .cdsinit file. For information about updating the .cdsinit file, see Modifying the .cdsinit File.

Register Area Estimators

Use the leRegAreaEstimator SKILL function to register area estimators.

leRegAreaEstimator      (s_functionSymbol;
        t_functionAlias;
        l_legalObjectTypes;
        [l_parameterNameValuePairs]; 
        [t_runMode];   
        );
     => t / nil

Specify the following parameters:

  • s_functionSymbol: Area estimation function name.
  • t_functionAlias: Alias name, which is a display name for the area estimation function. The alias name will be displayed in a cyclic list in the related forms.
  • l_legalObjectTypes: Legal object types for the function, which can be used as filters to list relevant area estimators. Valid Values: `cellview, `cluster, `group, `inst
  • l_parameterNameValuePairs: List of parameter name-parameter default value pairs, which might be required internally by the area estimator for area calculation formulae.
  • t_runMode: The run mode, which specifies the call mechanism. Valid Values: “direct”, “deferred”.
    • direct: The information required for area estimator to run is present and the area value can be viewed. For example: The input is a schematic cellview ID. The area estimator traverses the entire hierarchy to output an area.
    • deferred: You can set the area estimator and apply the parameters. This information is stored, to be called later when all the information is available. For example: The input is a physical layout cellview ID. Area calculation is deferred until the Generate From Source netlisting step. Once the physical hierarchy is available, area estimation is called.
      The sequence of arguments for registering an area estimator function must the same as defined in the area estimator function.

For more information, see leRegAreaEstimator.

Example 1:

Here, the areaEstUsingLayoutPRBoundary area estimator function is registered with alias estAreaUsingLayoutPR, cellView object type, parameter multFactor with default value 10, and the run mode set to direct.

leRegAreaEstimator('areaEstUsingLayoutPRBoundary 
        "estAreaUsingLayoutPR"
     '('cellView)
     '((multFactor 10))
     "direct"
     );

Example 2:

Here, the areaEstUsingLayoutBBox area estimator function is registered without any parameter.

leRegAreaEstimator('areaEstUsingLayoutBBox 
                   "estAreaUsingLayoutBBox" 
                   '('cellView) 
                   nil 
                   "direct"
        );

You can define area estimation functions in the .cdsinit file using the leRegAreaEstimator function. When Virtuoso is launched, these area estimation functions are automatically registered.

The registered area estimator registration functions are displayed with their alias names in the Area Estimator drop-down list in the Configure Physical Hierarchy form.

For more information about the form, see Configure Physical Hierarchy.

Example 3

In the following example, the areaEstSchematic area estimator function is registered with alias stdCellAreaEst and the cellView object type. Parameters include Mode, which is of the type list.

leRegAreaEstimator('areaEstSchematic
        "cellAreaEst"
     '('cellView)
     '(
      (Library     "myLib")
      (Cell          "myCell")
      (View        "layout")
      (Mode         '("Row1" "Row2" "Row3")) // list support
      (spacing       1.2)
      (X_Axis        1.08)
      (Y_Axis        1.632)
     )
     "direct")

The registered area estimator registration function is displayed in the Area Estimator drop-down list in the Configure Physical Hierarchy form. Click Estimate to display the Area Estimation Parameters form.

Example 4

In the following example, the myClusterAreaEstimator area estimator function is first defined, and then registered with the cluster object type.

procedure(myClusterAreaEstimator(clusterId 
   @optional (factor 10.0) (num 4) (text       
   "myFunc"))
   100
   )
leRegAreaEstimator('myClusterAreaEstimator 
     "forCluster" 
     '('cluster) 
     '((factor 10.0) (num 4) (text  
      "myFunc")) 
     "direct"
     )

Area estimator functions that are registered with the cluster object type are displayed in the Area Estimator cyclic list in the Create Cluster Boundary form. Use the options in this form dynamically fill the various parameters.

For more information about the form, see Create Cluster Boundary Form.

Format/Signature of Area Estimator

The format / signature of an area estimator should have the following important characteristics:

areaEstimatorName ( d_objectId
     @optional   (param1 defVal1) (param2 defVal2) ..
      )
     => areaVal / nil
  • d_objectId: A valid database object ID, such as a cellview, instance, or cluster ID
  • @optional (param1 defVal1)...: An optional list of name value pairs, where name is the parameter name and value is the default value, such as int, float, text, list, and boolean. On calling the area estimator function, the parameters in the list can be filled dynamically through a simple UI which will be displayed before execution of area estimator.
    Example 1:
    Consider the following registered area estimator function, for which the parameter list is defined in the estimator SKILL function and is also registered through leRegAreaEstimator([l_parameterNameValuePairs];):
    leRegAreaEstimator('areaEstUsingLayoutPRBoundary
         "estAreaUsingLayoutPR"
         '('cellView)
         '((multFactor 10))
         "direct"
         );
    => t | nil
    When this area estimator function is called and the Estimate button is clicked in the Configure Physical Hierarchy form, a UI is displayed, where the parameters can be entered dynamically.
    The return value is either a floating number or nil.



    Example 2:
    Consider the following area estimator function defined as follows:
 procedure(myClusterAreaEstimator(clusterId 
          @optional (factor 10.0) (num 4)(text 
          "myFunc"))
                 100
         )

The function is registered using:

 leRegAreaEstimator('myClusterAreaEstimator 
      "forCluster" 
      '('cluster) 
      '((factor 10.0) (num 4) (text "myFunc"))
      "direct"
      )

After registering the function, click the Estimate button in the Create Cluster Boundary form. A UI is displayed, where the parameters can be entered dynamically.

Other Utility Functions

Use the leUnRegAreaEstimator function to unregister area estimation functions for one or more object types. For information about the syntax and arguments, see leUnRegAreaEstimator.

Example

To unregister an area estimation function with "simpleAreaEst"alias name, which appears in the Area Estimator drop-down list for the cluster object type, use:

leUnRegAliasEstimator( "simpleAreaEst" ('cluster) )

Displaying Registered Area Estimators

leGetAreaEstimationDisplayNames

Use the leGetAreaEstimationDisplayNames function to retrieve the area estimation function alias name list defined for the specified object types. For information about the syntax and arguments, see leGetAreaEstimationDisplayNames.

Example

To retrieve the alias name list for area estimation functions with the cluster object type, use:

funcCallbackSymbol = leGetAreaEstimationDisplayNames( ('cluster) )

leGetAreaEstimatorFunction

Use the leGetAreaEstimatorFunction function to retrieve the function symbol of the area estimation function registered for the specified alias name and object type. For information about the syntax and arguments, see leGetAreaEstimatorFunction.

Example

funcCallbackSymbol = leGetAreaEstimatorFunction( "simpleAreaEst" ('cluster) )

leGetAreaEstimatorParamList

Use the leGetAreaEstimatorParamList function to retrieve the parameters of the area estimation function registered for the specified alias name and object type. For information about the syntax and arguments, see leGetAreaEstimatorParamList.

Example

funcParam List = leGetAreaEstimatorParamList( "simpleAreaEst" ("cluster") )

leGetAreaEstimatorRunMode

Use the leGetAreaEstimatorRunMode function to retrieve the Run Mode list of the area estimation function registered for alias name and object types.

Example

funcParam List = leGetAreaEstimatorParamList( "simpleAreaEst" ("cluster") )

The following SKILL script fragment uses the leGetAreaEstimatorFunction, leGetAreaEstimationDisplayNames, leGetAreaEstimatorFunction, leGetAreaEstimatorParamListz, and leGetAreaEstimatorRunMode functions.

foreach( d leGetAreaEstimationDisplayNames('('cellView))
         printf("Display Name = \"%s\", 
                 CallBack Name = \"%s\", 
                 Parameter List = %L, 
                 Run Mode = \"%s\"\n" 
   leGetAreaEstimatorFunction(d '('cellView)) 
   leGetAreaEstimatorParamList(d '('cellView)) 
   leGetAreaEstimatorRunMode(d '('cellView)))
  )

Sample Area Estimation Procedures

Procedure 1:

procedure(areaEstUsingLayoutPRBoundary(
     cv 
    @optional (multFactor 10.0) (pinArea 0.14) (numPins 10))
          let( ((area 0.0) ele master)
     foreach( ele cv~>instances
    master = ele~>master
    if(master~>prBoundary then
     area = area + master~>prBoundary~>area
           else
    area = area + abs((caadr(master~>bBox) - caar(master~>bBox))
    * (cadadr(master~>bBox) - cadar(master~>bBox)))
     )
    )
     area = area * abs(multFactor) + numPins * pinArea
    )
   )
