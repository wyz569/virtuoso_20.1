(ICADVM20.1 Only) In Layout EXL, Floorplan is available under the Plan menu.

autoGenHierPinOptMode

layoutXL.floorplan autoGenHierPinOptMode cyclic { "Top Down" | "Bottom Up" }

Description

(ICADVM20.1 EXL Only) Specifies the mode for running Pin Optimizer hierarchically on the blocks that were generated using the Auto-Generate Hierarchy form. The valid values are:

  • Top Down: Pin Optimizer is first run on the level-0 and level-1 soft blocks to get the best possible net length. Any existing pins, for example those of any level-1 hard blocks, are recognized and the soft block pins are aligned to these pins. Next, the pin positions of the level-1 soft blocks are used to derive the pin positions inside them. This process continues until the leaf level is reached.
  • Bottom Up: Follows the same behavior as described in Top Down, but runs in the opposite direction. Pin Optimizer is first run on pins of the leaf level layouts.

This option is effective only when autoGenHierRunPinOpt is set to t.

The default value is Top Down.

GUI Equivalent

Command

Floorplan – Auto Generate Hierarchy

Field

Run Pin Optimization (Auto-Generate Hierarchy)

Examples

envGetVal("layoutXL.floorplan" "autoGenHierPinOptMode")
envSetVal("layoutXL.floorplan" "autoGenHierPinOptMode" 'cyclic "Bottom Up")

layerChangeResize

layoutXL.floorplan layerChangeResize cyclic {"currentSize" | "lastLayerValue" | "minWidth"}

Description

Specifies the dimensions of pins when they are moved from one layer to another. The valid values are:

  • currentSize: The current pin size is retained. However, the pin is resized if the minWidth of the destination layer is higher than the current pin size.
  • lastLayerValue: This setting is applicable only to pins that were placed in the destination layer earlier within the current session. Their lastLayerValue is stored, and the pins are automatically resized to this value when moved back to these layers. If the value is not available, then the pin size is set to the higher of the two values - current pin size and the minWidth of the new layer.
    The lastLayerValue is stored only for the current session.
  • minWidth: The pin is resized to the minWidth of the destination layer.

The default value is currentSize.

GUI Equivalent

None

Examples

envSetVal("layoutXL.floorplan"  "layerChangeResize" 'cyclic "lastLayerValue")
envSetVal("layoutXL.floorplan"  "layerChangeResize" 'cyclic "minWidth")

layoutXL.floorplan offSetMode cyclic {"Auto" | "Custom"}

Description

Sets the pin offset mode to either Auto or Custom.

When set to Auto, the Pin Optimizer automatically places the first pin on a grid such that there is no spacing violation with the corner pins of the adjacent side. The Pin Optimizer also ensures that there is no wastage of grid space.

When set to Custom, you can manually set the pin offset values for all layers.
