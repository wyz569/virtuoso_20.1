Updating Pin Labels and Text Displays

During pin optimization, pins are re-positioned to obtain the shortest possible net length. However, the Pin Optimizer does not update pin labels and text displays for clear visualization. This may result in positioning of pin labels and text displays outside the prBoundary causing overlaps and inapt visualization, defeating the purpose of labeling.

Therefore, after running the Pin Optimizer, use the Floorplan > Label Update command to update pin labels and text displays for clearer visualization.

In Layout EXL, choose Plan – Pin Planning – Label Update.

Label Update GUI Options

The Floorplan > Label Update command displays the Label Update form.

The Floorplan > Label Update command is disabled when the top-level cellview is in the read-only mode.

The Label Update form has the following options for updating pin labels and text displays:

  • Mode: Choose one of the following modes:
    • All: Runs on all soft block pin labels and text displays in the current design.
      Soft blocks that are marked as read-only are make editable and a warning message is displayed in the CIW.
    • Selected: Runs only on the selected pin labels and text displays. You can select:
      • Top-level and level-1 pins
      • Individual soft blocks
      • Individual top-level pin labels and text displays
  • Adjust Label Orientation:
    • Make Label Perpendicular To Nearest Edge: Makes labels and text displays perpendicular to the nearest prBoundary edge. The nearest prBoundary edge is determined based on the minimum distance of the label’s parent pin center from the prBoundary.
      Orientation remains unchanged for labels and text displays that are already perpendicular to the nearest prBoundary edge.
  • Adjust Label Position: Includes the following options:
    • Pull Label Inside P&R Boundary: After adjusting the label orientation, some labels and text displays may protrude outside the prBoundary. Select such labels and text displays and choose the Pull Label Inside P&R Boundary option. When you choose this option, the tool tries different permutation and combinations for justification and orientation to place these labels and text displays inside the prBoundary. The label or text display origins, however, remain unaltered.
      If all labels and text displays could not be successfully pulled inside the prBoundary, an appropriate report is displayed in the CIW.
    • Move Label Origin to Pin Edge Nearest to Boundary: Irrespective of their initial origins, on selecting Make Label Perpendicular To Nearest Edge, all labels and text displays are moved to the center of the nearest pin edge as illustrated in the following figures:
      The Move Label Origin to Pin Edge Nearest to Boundary option is not selected by default.
    Depending on the access direction of the nearest prBoundary edge from the label parent pin, the orientation and justification of labels is set based on the following table:

    Edge

    Orientation / Justification

    Left

    R0/centerLeft

    Right

    R0/centerRight

    Top

    R90/centerRight

    Bottom

    R90/centerLeft

    The Adjust Label Orientation and Adjust Label Position options will not run for labels and text displays for which:
    • The label or text display origin is already outside the prBoundary.
    • The label or text display parent is outside prBoundary.

    In the above situations, appropriate message is printed in the report.
  • Adjust Label Size: Includes the following options to Resize Labels:
    • Update Label Size: Choose this option and specify the label size in the adjoining box.
      The Update Label Size option will not run in the following situations.
      • On floating labels and text displays that are not associated with any parent pin.
      • When the label or text display parent is not pin.
      • When labels or text displays for the instance pins are present on the top-level instead of within the same cellview as the pin.

      In the above situations, appropriate message is printed in the report.
    • Auto Reduce Label Size: Choose this option when label or text display overlaps exist even after making the above settings. This option intelligently computes the overlaps and reduces the size of the overlapping labels or text displays so as to remove any remaining overlaps. The size of labels or text displays is reduced upto a minimum threshold value as specified in the adjoining text box.

When multiple options in the Label Update form are selected, the following order of execution is followed for best results:

  1. Update Label Size
  2. Make Label Perpendicular to Nearest Edge
  3. Move Label Origin to Pin Edge Nearest to Boundary
  4. Pull Label Inside P&R Boundary
  5. Auto Reduce Size to Remove Overlaps
