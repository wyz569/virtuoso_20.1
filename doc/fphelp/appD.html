Floorplan Property File

You can define the soft block properties, such as boundary, pins, and halo blockage for a selected instance on the Instances tab in Configure Physical Hierarchy form. These properties are used by the Generate Physical Hierarchy command to create soft blocks for the instances in the physical hierarchy. The properties defined for a soft block are retained in one session. Once, the Virtuoso session is closed, the defined properties are erased from the memory. In such cases, Floorplan property file is used for the persistent storage of the soft block properties, which can be reloaded when you work with a new Virtuoso session.

You can save the soft block properties in a property file by selecting File -> Save Floorplan File in the Configure Physical Hierarchy form.

The syntax of the Floorplan property file is as follows:

Top-Level
(
Master properties.....
 Boundary tab properties....
  (Terminal properties.............
   Term props1....
   Term props2....
   ...........
   Term propsN....
  )
  (Halo properties............)
)

Binding Properties

The binding properties provide the binding information for the soft block. The binding properties are defined in the CPH form as shown in the following figure.

The following table lists the binding properties along with their description.

Property Name GUI Option Name Description

libName

Physical Library

Name of the layout library to be generated

cellName

Physical Cell

Name of the layout cell to be generated

viewName

Physical View

Name of the layout view to be generated

schLibName

Library (tooltip)

Name of the schematic library used to read properties.

schCellName

Cell (tooltip)

Name of the schematic cell used to read properties

schViewName

View (tooltip)

Name of the schematic view used to read properties

inhViewList

Inherited View List

Specifies the design views that will be used to descend into the hierarchical design to find stop views.

inhStopViewList

Inherited Stop List

While descending the hierarchical design it stops if it finds any of the views, such as layout or abstract. The view list then populates the CPH form with its properties.

stopString

NA

These views are searched for IO pad cells to create a pin list from the main list, which is filtered.

This property is kept for legacy reasons to support commands for backward compatibility in Virtuoso and will be deprecated in the next release.

hierPath

Name (the complete hierarchy path)

This is the complete hier path of an instance with respect to the top-level cellview.

versionName

NA

The default value for this is an empty string (“ “).

This property is kept for legacy reasons to support commands for backward compatibility in Virtuoso and will be deprecated in the next release.

blockType

Block Type

Describes the type of block, whether “CUSTOM” or “DIGITAL”.

The three properties symAbst, block, and abgen are boolean properties, which control the soft abstract generation of the block. If symAbst is set to true and block and abgen are set to false, then a soft abstract is created. However, if symAbst is set to false and block and abgen are set to true, then a soft abstract is not created.

symAbst

Type

Specifies whether or not we have to create a soft abstract of the block. This property is mostly specified in cases where the instance in schematic is symbol.

block

Type

If this property is specified as NIL, then it creates a soft abstract.

This property would be deprecated in the next release.

abgen

Type

If this property is specified as NIL, then it creates a soft abstract.

This property would be deprecated in the next release.

Boundary Tab Properties

The boundary tab properties are defined to customize the prBoundary created in the layout. For more information, refer to Figure 1.

Figure 1

The boundary tab properties are defined in the Boundary tabbed page of the CPH form as shown in the following figure.

The following table lists the boundary properties along with their description.

Property Name GUI Option Name Description

boundaryPolyType

radio buttons in the Boundary tabbed page

Select the type of boundary to be created. The boundary type can be ‘rectangular’ or ‘polygon’.

cyclic1

combo box to select among the various options

This property allows you to specify one of the area calculation mode. The four options in this property are: Height, Width, Area Utilization, and Aspect ratio. Each option has a value associated with it, which ranges from 1 to 4, where 1 represents Area Utilization, 2 is for Aspect Ratio, 3 for Width, and 4 is for Height.

cyclic2

combo box to select among the various options

This property allows you to specify one of the area calculation mode. The four options in this property are: Height, Width, Area Utilization, and Aspect ratio. Each option has a value associated with it, which ranges from 1 to 4, where 1 represents Area Utilization, 2 is for Aspect Ratio, 3 for Width, and 4 is for Height.

boundaryHeight

value of boundary height in any cyclic field

Specifies the height of the prBoundary.

boundaryWidth

value of boundary width in any cyclic field

Specifies the width of the prBoundary.

areaUtil

value of area utilization in any cyclic field

Defines the value of area utilization. For example, if utilization is 25%, then accordingly it will calculate the prBoundary area.

aspRatio

value of aspect ratio in any cyclic field

This is the ratio of width/height values. For example, if aspect ratio is 2, then Width = 2*Height.

areaMode

Mode (‘Manual’, ‘Avg Area Per Gate’, Use Estimator)

Describes the area calculation mode, whether Manual, areaPerGate, useEstimator. By default, the area mode is selected as Manual. This is populated in the Boundary tab of the CPH form. For more information, refer to Figure 1.

area

Area field

This property is used to provide the fixed value of prBoundary area.

gatecount

Gate Count field

This option is mostly used by digital users. It specifies the number of gates. This is in unison with avgAreaPerGate.

avgAreaPerGate

Avg. Area Per Gate field

This option is usually used by digital users. You can specify the value for gate area. This is in unison with gatecount.

areaCallBack

Area Estimator cyclic field

The name of the area estimator function, which is used to calculate prBoundary. For more information on area estimator, refer to the Virtuoso Layout Suite XL User Guide.

polygonList

X and Y coordinate columns

The list of polygon points for prBoundary. This is populated in the Boundary tab of the CPH form.

Terminal Properties

The terminal properties describes the properties that are to be defined for a terminal. The properties that are defined are as displayed in the following diagram.

The following table lists the terminal properties along with their description

Property Name GUI Option Name Description

termCreate

NA

This is a boolean value, which indicates whether or not to create a terminal.

netName

Net Name

Name of the net.

termName

Term Name

The name of the terminal.

termLayer

Layer and initIOPinLayer environment variable

The name of the layer on which the pin figure has to be created. The valid values are the layers present in the technology data.

termPurpose

Layer and initIOPinLayer environment variable

This specifies the layer purpose on which the pin figure has to be created. The valid values are the purposes present in the technology data.

termDirection

Term Type

Describes the direction of the terminal. The valid values are: “input”, “output”, “inputOutput”, “switch”, “jumper”, and “unused”.

termSigType

Sig Type

Determines the signal type of the terminal. The valid values for the signal type are: "signal", “ground”, "supply", "clock", "analog", "tieOff", "tieHi", "tieLo”, "scan", and "reset".

termCriticality

Criticality

Determines the criticality of the terminal. The valid value ranges from -128 to 128.

termNum

Number

Defines the number of pins to be created on a terminal. The valid range is from 1 to 20.

termHeight

Height

Defines the height of the pin to be created on a terminal. The value values are all positive integer values.

termWidth

Width

Defines the width of the pin to be created on a terminal. The value values are all positive integer values.

The following table lists the halo properties along with their description

Property Name GUI Option Name Description

haloLayer

Layer

The name of the layer on which halo blockages are to be created.

haloBlockageType

Blockage Type

Describes the type of blockage. The valid values for the blockage type are: “placement”, “routing”, “fill”, “slot”, “pin”, “feedthru”, and “screen”. The default value is “routing”.

For placement blockage, the halo layer is NIL.

Offsets

Left

Describes the distance of a halo blockage from the left of the prboundary. The valid value is either 0 or a positive float value.

haloRightOffset

Offsets

Right

Describes the distance of a halo blockage from the right of the prboundary. The valid value is either 0 or a positive float value.

haloBottomOffset

Offset

Bottom

Describes the distance of a halo blockage from the bottom of the prboundary. The valid value is either 0 or a positive float value.

haloTopOffset

Offset

Top

Describes the distance of a halo blockage from the top of the prboundary. The valid value is either 0 or a positive float value.

haloType

Blockage Type

Describes the type of halo. The valid values for the halo type are placement and layer halo.
