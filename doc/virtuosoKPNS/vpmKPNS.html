Virtuoso Known Problems and Solutions
Product Version ICADVM20.1, October 2020

Virtuoso Power Manager

This document describes the known issues with Virtuoso® Power Manager and suggests the workarounds for these issues. Each issue is identified by a Cadence Change Request (CCR) number.

Related Documentation

  • Virtuoso Power Manager User Guide

CCR 2325180: Power Manager environment variable management as per new ICADVM20.1 strategy

Description: The following environment variables should not be used from .cdsenv because they have no relevance to Power Manager.

lp allowDomainWithoutElements boolean nil nil

lp prepareCLPOutputDir string ".cadence/dfII/1801" nil

lp runLowPowerChecks boolean nil nil

lp inputVoltageToleranceLowerBound float -0.1 nil

lp inputVoltageToleranceUpperBound float 0.1 nil

lp inputGroundVoltageToleranceLowerBound float -0.1 nil

lp inputGroundVoltageToleranceUpperBound float 0.1 nil

lp flagPotentialLSViolations boolean nil nil

lp missingLSCheckSeverity cyclic "error" nil ("error" "warning" "ignore")

lp missingISOCheckSeverity cyclic "error" nil ("error" "warning" "ignore")

lp incompatibleLSCheckSeverity cyclic "error" nil ("error" "warning" "ignore")

lp redundantLSCheckSeverity cyclic "warning" nil ("error" "warning" "ignore")

lp redundantISOCheckSeverity cyclic "warning" nil ("error" "warning" "ignore")

lp reportLPRCViolationsInAllModes boolean nil nil

Solution: This issue has been documented for your information only. There is currently no workaround available.
