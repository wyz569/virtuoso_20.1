Analog Placer Functions (IC6.1.8 Only)

This section provides syntax, descriptions, and examples for the Cadence® SKILL functions associated with the analog placer.

Only the functions documented in this chapter are supported for public use. All other functions, regardless of their name or prefix, and undocumented aspects of the functions described below, are private and are subject to change at any time.

nclAnalogAdjCellPins

nclAnalogAdjCellPins(
[ d_cellViewID ]
)
=> t / nil

Description

Adjusts the pins in the specified cellview to be close to the terminals to which they are connected, while also respecting any constraints set on the pins. If you do not specify a cellview, the function operates on the currently open cellview.

Arguments

d_cellViewID

Database ID of the cellview.

Value Returned

t

The pins were adjusted successfully.

nil

Pin adjustment failed.

Example

Adjusts the pin locations on the currently open cellview.

nclAnalogAutoPlaceCB

nclAnalogAutoPlaceCB(
[ d_cellViewID ]
[ g_effortLevel ]
[ g_updatePRBoundary ]
)
=> t / nil

Description

Opens the Analog AutoPlace form to run Analog Automatic Placement, which optimizes the placement of instances and pins to meet any constraints set on the design, while also optimizing the design for both area and wire length. If you specify d_cellViewID, Analog Automatic Placement runs automatically without displaying the form.

Arguments

d_cellViewID

Database ID of the cellview.

g_effortLevel

Effort level for the analog auto placement.

Valid values are 1 (minimal effort); 2 (nominal effort); 3 (maximum effort).

g_updatePRBoundary

Updates the place and route boundary after placement has run. When set to nil, the place and route boundary is considered fixed and is not updated.

Value Returned

t

The Analog AutoPlace form is displayed or, if d_cellViewID is specified, Analog Automatic Placement runs automatically.

nil

The form was not displayed or Analog Automatic Placement was not run.

Example

Displays the Analog AutoPlace form.

nclCreateXSym

nclCreateXSym (
d_cv
[ d_net1 d_net2]
)
=> l_figGroupsCreated

Description

This function creates either a single figGroup or all of the needed cross symmetric figGroups depending on the options provided to allow cross symmetric routing.

Arguments

d_cv

Specifies the cellview in which the cross symmetric figGroup is to be created.

d_net1 d_net2

Specifies the nets to create a cross symmetric figGroup.

Example

nclCreateXSym(cv)

Creates all of the cross symmetric figGroup geometry needed to allow cross symmetric routing based on the constraints, devices, and pins in the design.

nclCreateXSym(cv netA netB)

Creates a cross symmetric figGroup for the two specified nets if they have a symmetry constraint between them.

nclRegPostPlacementTrigger

nclRegPostPlacementTrigger(
S_triggerFuncName
[ x_priority ]
)
=> t / nil

Description

Registers a function to be called after analog placement is complete. The registered functions should take only one argument specifying the cellview that was placed. The functions are called in the specified order (lowest to highest priority) after any automatic or batch analog placement operations are finished.

Arguments

S_triggerFuncName

Name of a registered post placement trigger function.

x_priority

Order in which the trigger functions are run (lowest to highest). If no priority is specified, the functions are run in the order they are listed.

Value Returned

t

The function was registered.

nil

The function was not registered.

Example

procedure(postPlacementTrigger(cellView)
  printf("PostPlacementTrigger called on cellView: %L\n"
list(cellView~>libName cellView~>cellName cellView~>viewName))
)
nclRegPostPlacementTrigger('postPlacementTrigger 10)

nclSetAspectRatioRange

nclSetAspectRatioRange(
d_cv
f_minAspectRatio
f_maxAspectRatio
[ f_overrideArea ]
)
=> d_cbc / nil

Description

Modifies the cell boundary constraint on the prBoundary of the specified cellview by specifying the minimum and maximum acceptable aspect ratios. A new cell boundary constraint and prBoundary is created, if required.

Arguments

d_cv

Cellview ID to which the prBoundary that is referenced by the cell boundary constraint belongs.

f_minAspectRatio

Minimum acceptable aspect ratio for the specified cellview

f_maxAspectRatio

Maximum acceptable aspect ratio for the specified cellview

f_overrideArea

The required area of the cellview. If this value is not specified, then the cell boundary constraint uses the current area of the cellview.

Value Returned

d_cbc

ID of the cell boundary constraint that was created or modified.

nil

The cell boundary constraint could not be created or modified

Example

cvParent = geGetEditCellView()
nclSetAspectRatioRange(cvParent 0.5 2.0)
cvChild = dbFindAnyInstanceByName(cvParent  "I0")->master
nclSetAspectRatioRange(cvChild 0.25 4.0 30000.0)
