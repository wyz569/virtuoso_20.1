Virtuoso Placer Functions (ICADVM20.1 EXL Only)

This section provides syntax, descriptions, and examples for the Cadence® SKILL functions associated with the Virtuoso Placer.

Only the functions documented in this chapter are supported for public use. All other functions, regardless of their name or prefix, and undocumented aspects of the functions described below, are private and are subject to change at any time.

lobGetRegisteredAdjacentFillDefsProc

lobGetRegisteredAdjacentFillDefsProc(
)
=> t_functionName / nil

Description

(ICADVM20.1 EXL Only) Returns the name of the function that is registered as the adjacent fill definition procedure.

Arguments

None

Value Returned

t_functionName

The name of the function that is registered as the adjacent fill definition procedure.

nil

The command was unsuccessful.

Example

In the following example, cdn20FFAdjacentFillDefs stores a user function symbol. This is passed as an argument in lobRegisterAdjacentFillDefsProc to register the adjacent fill definition procedure. Then, lobGetRegisteredAdjacentFillDefsProc is run to determine the name of the function that is registered as the adjacent fill definition procedure:

procedure( cdn20FFAdjacentFillDefs() 
 ptapfill = '(nil
  name "ptapfill"
  diffusionLPP ("Active" "drawing")
  fingerLPPs ( ("Poly" "drawing") )
  libName "cdn20FF"
  cellName "ptap"
  viewName "layout"
     type      "instance"
 )
 ntapfill = '(nil
  name "ntapfill"
  diffusionLPP ("Active" "drawing")
  fingerLPPs ( ("Poly" "drawing") )
  libName "cdn20FF"
  cellName "ntap"
  viewName "layout"
     type      "instance"
 )
 i = 2
 list(ntapfill ptapfill)
)
lobRegisterAdjacentFillDefsProc('cdn20FFAdjacentFillDefs)
lobGetRegisterAdjacentFillDefsProc()
=> "cdn20FFAdjacentFillDefs"

lobGetRegUserProc

lobGetRegUserProc(
g_userProc
)
=> l_userProcsString / nil

Description

(ICADVM20.1 EXL Only) Returns a list of strings that contain the name of the specified registered user procedure.

Arguments

g_userProc

Specifies the name of the registered user procedure to filter strings. Valid symbols are: ‘lobGetAdjacentFillDefsProc, ‘lobGetTransitionFillDefsProc, and ‘lobDummyNetNameProc.

Value Returned

l_userProcsString

A string or a list of strings containing the name of the registered user procedure.

nil

The command was unsuccessful.

  viewName "layout"
     type      "instance"
 )

  fingerLPPs ( ("Poly" "drawing") )

lobRegUserProc

lobRegUserProc(
g_userFunction
)
=> t / nil

Description

(ICADVM20.1 EXL Only) Registers a user-defined function for a specific purpose, as needed by layout objects. This function is currently used to register procedures to retrieve adjacent fill definitions.

Arguments

g_userFunction

Specifies the symbol of the function to be registered. Valid values are: @lobGetAdjacentFillDefsProc, @lobGetTransitionFillDefsProc, and @lobDummyNetNameProc.

Value Returned

t

The key parameters are passed and the function is registered.

nil

The key parameters were not passed.

Example

In the following example, a user-defined function cdn10GetAdjacentFillDefs is registered.

lobRegUserProc(?lobGetAdjacentFillDefsProc 'cdn10GetAdjacentFillDefs)
