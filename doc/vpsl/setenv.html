Setting Up the Environment Variables

You can set a few environment variables before invoking the EM/IR software to pre-fill the fields of the IR (Voltage) Drop Analysis and the Electromigration (EM) Signal Analysis forms.

A few examples of environment variables are:

  • setenv _vsaIRDropSimDir ./simulation
    Specifies the location of the Spectre simulation directory in the IR Drop Analysis form.
  • setenv _vsaEMSimDir ./simulation
    Specifies the location of the Spectre simulation directory in the Electromigration Power Analysis form.
  • setenv _vsaEMQRCDir ./qrcDir
    Specifies the location of the QRC run.
  • setenv _vsaEMDataFile ./emDataFile.txt
    Specifies the location of the emDataFile. This variable is optional. If it is not set, the software looks for a file called emDataFile.txt first in the current working directory and then in the DFII technology directory.
  • setenv _vsaEMTechFile path_of_qrcTechFile
    Specifies the location of qrcTechFile. If emDataFile is provided, this variable is ignored.
  • setenv _vsaEMMapFile path_of_mapfile
    Specifies the location of the mapfile that contains layer mapping information between ICT/p2lvsfile/DFII layer names when vsaEMTechFile is specified. This variable is optional. If not specified, the tool assumes that the ICT/p2lvsfile/DFII layer names are same.
  • setenv _vsaEMDeltaT 10.1
    Specifies the deltaT value for EM analysis. It is used to specify maximum rise in temperature in degree Celsius, caused due to Joule Heating. It must be a positive value.
    If you do not specify the rise in temperature, Power IR/EM considers a default temperature of 5 degree Celsius.
  • setenv _vsaEMPreExecuteFile “test.il”
    Specifies the pre execute file for EM analysis in qrcTechFile flow.
  • setenv _vsaP2Lvs ./p2lvsfile
    Specifies the location of the p2lvsfile used during RCX extraction.
  • setenv _vsaEMColumnViewConfigFile ./filename
    Specifies the location of the column selection configuration file for displaying customized EM analysis results.
  • setenv _vsaSaveOnlyFailedResults true
    When this variable is set to “true”, the software saves EM analysis results for only the failed nets. This setting overrides the default selection of the Save Only Failed Results field in the Save Electromigration Analysis to a File form.
  • setenv _vsaLoadOnlyFailedResults true
    When this variable is set to “true”, the software loads EM analysis results for only the failed nets. This setting overrides the default selection of the Load Only Failed Results in the Select Existing Results File form.
  • setenv _vsaSelectLayers “Via1 m.*”
    Specifies the list of selected layers for EM analysis.
  • setenv _vsaSelectResistors “Rh.* Re.*”
    Specifies the list of selected layers for EM analysis.
  • setenv _vsaIgnoreLayers “Via1 m.*”
    Specifies the list of layers to be ignored for EM analysis.
  • setenv _vsaIgnoreResistors “Rh.* Re.*”
    Specifies the list of resistors to be ignored for EM analysis.
  • setenv _vsaUseNewLayers false
    When this variable is set to false, Power IR/EM uses the layer names specified in the technology file for highlighting the IR/EM analysis results. By default, this variable is set to “true”. When set to “true”, the tool uses the layer names vavo0 through vavo8 for highlighting the IR/EM analysis results.
  • setenv _vsaDrfFile “vavo.drf”
    Specifies the layer names to be highlighted in the IR/EM analysis results. If this file is not specified, the tool will first look for the file in the current directory and then in the Power IR/EM Installation.
    The new layer names to be highlighted should be specified as vavo0 through vavo8.
  • setenv _vsaTechFile “vavo_techfile”
    Specifies the display properties for the layers to be highlighted in the IR/EM analysis results. If this file is not specified, the tool will first look for the file in the current directory and then in the Power IR/EM Installation.
    The above variables, _vsaDrfFile and _vsaTechFile, are specified together. Refer to “Technology File and Display Resource File ASCII Syntax Reference Manual” for more information about the contents of these files and how to modify them.
  • setenv _vsaICTConductorUnit “mA”
    Specifies the unit for current density of metal layers as mA. The ICT file supports A/cm2 and mA/um as the units for specifying the current density of metal layers. You can use this variable to specify the current density of these layers in mA.
  • setenv _vsaEnablePWL “1”
    Specifies that the piece-wise linear (PWL) rule will be applied to the intermediate area values.
  • setenv _vsaSupport28nmRules “true”
    Enables explicit support for 28-nm rules. By default, the tool supports 28-nm rules when the conditions specific to 28 nm are specified in the emDataFile. This variable is helpful in case of a custom function where rules are not specified in the emDataFile.
  • setenv _vsaTechnologyNode 40
    Enables you to configure technology-dependent calculations. For example effective length calculation. You can also specify the technologyNode = 40 in the emDataFile.
  • setenv _vsaEnablePowerGridRules “true”
    Enables power-rail analysis. By default, power-rail analysis is disabled.
  • setenv _vsaLinearInterpolation “1”
    Enables linear interpolation for the temperature derating factor.
  • setenv _vsaDisableAllDesignResistorAnalysis “true”
    Disables analysis of all design resistors that are not connected to the selected pins. By default, the software analyzes all design resistors, irrespective of whether they are connected to the selected pins or not.
  • setenv _vsaLSFMachineString “bsub -q lnx64”
    Specifies the name of the LSF command string that will launch Power IR/EM in the batch mode.
  • setenv _vsaApplyRThreshold “0.05”
    Specifies the threshold ratio “r” for AC-Peak analysis. The default value of threshold ratio is 0.05. You can also specify the threshold ratio in the following ways:
    • In the Electromigration Edit/View Custom Parameters form, by using the applyRThreshold custom variable
    • In the ICT file, by using the applyR keyword

    For details, see Rules for Defining Limit-Based Parameters in the “EM Rules Specification” chapter.
  • setenv _vsaRMSMultiplierPin “1.0”
    Specifies the RMS relaxation factor for EM power-grid analysis. The default value is 1.0. You can also specify the value in the Edit/View Custom Parameters form by selecting the RMSMultiplierPin parameter.
  • setenv _vsaRMSMultiplierSignal “1.0”
    Specifies the RMS relaxation factor for signal EM analysis. The default value is 1.0. You can also specify the value in the Edit/View Custom Parameters form by selecting the RMSMultiplierSignal parameter.
  • setenv _vsaDisableLastConfig “true”
    Specifies that the configuration file created in the previous session will not be loaded by default for either IR drop or EM analysis in the current run. When you open either the IR (Voltage) Drop Analysis form or the Electromigration Power Grid Analysis form, the fields are prefilled with the settings specified in the previous session. These settings are saved in the vsaIRLast.cfg file for IR drop analysis and in the vsaEMLast.cfg file for EM analysis.
  • setenv _vsaUncheckViewFile “true”
    Specifies that files generated during both IR drop and EM analysis should not open for viewing by default. In the GUI, the View file option to view the command files and result files for both IR and EM analyses is checked by default. You can specify this variable if you do not want to view these files.
  • setenv _vsaJmaxLife “5.0”
    Specifies the scaling factor that applies to the current density limits for different lifetimes. The unit of the lifetime value is the same as that specified using the em_lifetime_units parameter defined in the process section of the ICT file. The default is 5.0. You can also specify the lifetime value in the Electromigration Edit/View Custom Parameters form by selecting the JmaxLife parameter.
  • setenv _vsaMinWarning “true”
    Specifies that minimum warning messages will be given during a session. When this environment variable is set to true, the following warning messages will be suppressed:
    • Duplicate via width information warning message while specifying the emDataFile flow in EM analysis. By default, the via width information in the emDataFile will override that specified in qrcTechFile.
    • Missing current density specifications warning message
  • setenv _vsaIncludeDFIINodeCol “true”
    Specifies that columns providing information about the DFII node 1 and DFII node 2 should be displayed in the Electromigration Analysis Results form. By default, information about these nodes is not displayed in the form. When this variable is set to true, the Customize display of EM results form shows DFII node 1 and DFII node 2 as options available in the Columns section. You can select the options to display these columns in the Electromigration Analysis Results form.
  • setenv _vsaDisableAutoArchive “true”
    Specifies that the IR drop analysis and EM analysis result files (IRDrop.results and EM.results) and form configuration files (IR.cfg and EM.cfg) should not be saved in the working directory. These files are saved in the working directory by default when you click the OK or Apply buttons in the IR Drop Analysis or the Electromigration Analysis forms.
  • setenv _vsaDBRefPinThreshold value
    This variable is only used while performing IR drop peak analysis in the vavo-db flow. It is used to specify the maximum value of variation allowed in the reference pin’s voltage value (difference between the min and max values). Pins that have a min-max voltage value variation less than the specified value will be considered stable and will be included in the IR drop peak analysis while the pins that have a min-max voltage value variation greater than the specified value will be excluded from the IR drop peak analysis.
    The range of threshold voltage variation values that can be specified is from 0 to 1.
    When “0” is specified, then those pins that have any variations in the min-max voltage values, will be excluded from the IR drop peak analysis.
    When “1” is specified, all pins will be included in the IR drop peak analysis irrespective of the min-max voltage voltage value variations.
    The default value of this variable is 0.05, which means the maximum allowed variation in the reference pin's voltage is 5% of its value. The software will include pins with a value less than 5% in the analysis and it will ignore all pins that have a min-max variation of greater than 5% and exclude them from the analysis.
    For IR drop peak analysis on transient data for dynamic pins, vavo-db simulation data is not recommended. Use the psf data instead. However, if you still want to do the IR drop peak analysis in the vavo-db flow, use the _vsaDBRefPinThreshold variable to define the threshold value for the stability checks.
    Dynamic pins have different min-max pin voltage values for a given period of analysis. For example, in transient analysis, the power supply pin and ground pin may be static pins, while the input and output pins may be dynamic pins.
  • setenv _vsaMicronUnit “true”
    Specifies that the units for length/width information in the extview.tmp file will be considered as micron.
  • setenv _vsaDumpResultDBPath path
    Specifies the path of the VSA_RESULTS directory that is used for viewing the IR drop and EM analysis results in vsaPlot. By default, this directory is created in the current working directory.
  • setenv _vsaBulkTerminal “r1,r2,..”
    This variable is only applicable to the design resistor flow. It is used to specify the bulk terminals of the design resistors that are to be ignored while performing EM analysis. You can provide the names of multiple terminals enclosed in commas. When this variable is specified, the value of the BulkTerminal parameter in the GUI is updated.

Manual Net Name Resolution

Power IR/EM automatically resolves the net name format used in the PSF/PSF-XL waveform or SST2 database format while importing simulation results. However, sometimes Power IR/EM might fail to correctly identify the net name convention and that will result in an error reading each net while importing the simulation results.

The environment variable _vsaNetNameFormat can be used to manually set alternative net name formats to use when importing the simulation results. The possible values of _vsaNetNameFormat are:

SpicePSF, Spice, SpectreSST, Spectre

This environment variable can be used as follows:

  1. When encountering excessive net names errors while importing the simulation results, check the currently used name format by typing the following in the Command Interpreter Window (CIW):
    _vsaGlobalIRDropNameFormat
    The variable and its current value is displayed in the CIW as follows:
    _vsaGlobalIRDropNameFormat “Spice”
  2. Set the _vsaNetNameFormat environment variable to a net name format other than the one returned in Step 1 above. Type the following in the CIW:
    setShellEnvVar("_vsaNetNameFormat=SpicePSF")
    To check the value of the _vsaNetNameFormat environment variable, type the following in the CIW:
    getShellEnvVar("_vsaNetNameFormat")
  3. Now repeat the IR or EM analysis to see if Power IR/EM can import the simulation results. If you still have problems importing the simulation results, you can change the value of _vsaNetNameFormat to one of the remaining values.
    When you have discovered the value of _vsaNetNameFormat that works for your design, set the environment variable in your setup file so that it will be used whenever you run Power IR/EM.
