Overview

Before running Power EM analysis, the EM reliability rules need to be specified. For example, the rule specifying different current densities with respect to width, length, and temperature.

These rules can either be read from the QRC technology file (qrcTechFile) or from the emDatafile, which includes user-specified rules.

For more information about the syntax of limit-based EM models, refer to the EM_Models section in the “Creating the ICT File” chapter in QRC Techgen Reference Manual.

Reading EM Rules from qrcTechFile

Currently, Power IR/EM only supports parameters for limit-based EM analysis.

For more information about the syntax of limit-based EM models, refer to the section titled, EM_Models, in the “Creating the ICT File” chapter in QRC Techgen Reference Manual.

Supported Limit-Based Parameters

The following parameters are supported:

  • em_vcwidth <value>
    Optional parameter for via layers that defines the width of an individual standard via layer in microns. This parameter is similar to the viaWidthList parameter in emDataFile.
  • em_W_n
    Optional parameter that specifies the wire width in microns below which a line is considered narrow. All lines are considered wide unless you specify otherwise using this parameter.
    It helps to select the rule between em_jmax_*_w and em_jmax_*_n. If w < em_W_n , then em_jmax_*_n is selected, otherwise em_jmax_*_w selected.
  • em_segment_length
    A process definition parameter that determines how to calculate the length of a wire segment. This parameter determines whether the segment length is of the type perimeter or centerline. The difference between the perimeter and centerline segment length types is as follows:
    • perimeter—specifies that the length calculation is along the wire outer edge boundary. This value is used for 28nm process nodes.
    • centerline—specifies that the length is extended to the edge of the segment along the centerline path. This value is used for 20nm process nodes.

    The following example shows how to use the em_segment_length parameter:
    em_segment_length centerline
  • em_recover
    Optional parameter that specifies the recovery factor used for calculating the average current density with recovery. You can set the recovery factor to -1.0 when you want to account for the average of the absolute current (avg(abs(current))) so that the negative current does not cancel the positive current values in arriving at the average current result.

Supported JMAX Parameters

For current density limit (jmax) analysis, the following parameters are supported:

  • em_jmax_dc_avg*
    Optional parameter that specifies the AVG analysis in Power IR/EM.
  • em_jmax_dc_peak*
    Optional parameter that specifies PEAK analysis in Power IR/EM.
  • em_jmax_dc_rms*
    Optional parameter that specifies DC RMS analysis in Power IR/EM.
  • em_jmax_ac_rms*
    Optional parameter that specifies AC RMS analysis in Power IR/EM.
  • em_jmax_ac_peak*
    Optional parameter that specifies AC_PEAK analysis in Power IR/EM.
Star ‘ * ’ implies that ‘_w’ and ‘_n’ rules are also supported.
jmax parameter, em_jmax_ac_rec* is not currently supported in Power IR/EM.

Rules for Defining Limit-Based Parameters

Below is an example of the EM rule for defining the em_jmax_ac_avg parameter.

[em_jmax_ac_avg <value> | <value_1> <area_1/width_1> [...] | EQU <fn(E)>]
[jmax_factor <temp1> <scale1> [<temp2> <scale2> ...]]
[jmax_life <life1> <scale1> [<life2> <scale2>.......]]
[current_direction up|down|both] 
[via_range <value>]
[conditions]
[single]
[power_rail]

All limit-based parameters are defined with the same structure but apply to different characteristics of the em_model (peak current versus average for instance). The following rules apply to all the limit-based em_model parameters:

  • Equations in EM Model
    An end of line (EOL) or one of the qualifiers (jmax_factor, current_direction, single, or via_range) marks the end of an equation specifying the current density limit. If the equation resolves into two separate equations, this will result in an error.
    For EQU, the following operators (case-insensitive) are supported:
    • SQRT
    • LOG
    • EXP
    • * / + -
    • ()
    • e
    • ^
      Square brackets ‘[ ]’ are not supported. A comma ‘,’ implies multiple equations, and results in an error. The period ‘.’ is ambiguous and should not be used to indicate multiplication (use * instead).
    Spaces are permitted to make an equation more readable.
    Variables within equations (such as deltaT, w, and l) will be populated by the analysis tool at run-time. Use the em_variables parameter in the process definition to globally declare variables that are used within the various em_models.
    The equation definition ends when reaching the EOL or another keyword (jmax_factor, current_direction, single, or via_range).
    Order Dependency
    The actual current density limit (value, PWL pairs, or equation) should follow immediately after the current density keyword (em_jmax_ac_avg). Any of the qualifiers (jmax_factor, current_direction, single, or via_range) will come after that. The order of the qualifiers is not important.
  • Units of Values
    The unit should be specified in accordance with the setting defined by the em_conductor_unit, em_via_unit, and em_via_area_unit parameters defined in the process definition. Units are specified as:
    • For a single value or an equation:
      • A/cm2 or mA/um for metal layers (width based)
      • A/via or mA/via for via layers (per via)
        You can also specify the current density of metal layers in mA instead of in A/cm2 or mA/um by using the environment variable,_vsaICTConductorUnit.
    • For a PWL table:
      • A/cm2 or mA/um, with the width specified in microns for metal layers
      • A/cm2 or mA, with the area specified in square microns for via layers
        By Default, PWL points are discrete points and not a continuous line. Therefore, the area values that fall between the two points do not match the rule. To apply PWL rules to intermediate area values, use the environment variable, _vsaEnablePWL. In this case, the tool interpolates the value for the intermediate area. For area values above the maximum area defined in the PWL, the tool will do constant interpolation. For area values below the minimum area defined in the PWL, the tool will do linear interpolation from <0 0> to the minimum value pair that is specified.
  • JMAX Factors
    jmax_factor <temp1> <scale1> [<temp2> <scale2> ...]
    jmax_factor is the optional scaling factor to use at different temperatures as compared to the reference temperature (defined by em_tref in the process definition). The temperature for jmax_factor should be specified in degree Celsius.
    Scaling factor is a positive integer: >1 to scale up, <1 to scale down, or 1 for no scale effect.
    When setting scale factors for multiple temperatures, they should be specified in an ascending sequence to enable interpolation.
  • JMAX Life
    jmax_life <life1> <scale1> [<life2> <scale2>.......]
    jmax_life provides the ability to set the scaling factor that applies to the current density limits for different lifetimes. Based on the lifetime value that you specify (using the em_lifetime_units parameter defined in the process section of the ICT file), the appropriate multiplication factor is applied to the nominal current density limit.
    For more information, see the Rules for Defining Limit-Based Parameters section in the “Creating the ICT File” chapter in QRC Techgen Reference Manual.
    You can specify the lifetime value by using the JmaxLife custom variable in the Electromigration Edit/View Custom Parameters form.
    You can also specify the lifetime value using the environment variable, _vsaJmaxLife.
  • Format of the Conditions
    Condition = [L/W/Lu/Wu/Lb/Wb/Lv2v/Td/r < | <= | > | >= <value in microns>] | [Itolerance < | <= | > | >= <value in percentage>]
    Conditions = [condition]*
    The L parameter specifies the length-based jmax values in microns.
    The W parameter specifies the width-based jmax values in microns.
    L and W are applicable for both metals and vias.
    Lu, Lb, Wu, and Wb are applicable only for vias.
    Lv2v and Itolerance are applicable for power_rail only.
    Where,
    L = length W = Width Lu = Upper metal length Lb = Bottom metal length Wu = Upper metal width Wb = lower metal width Lv2v = Distance between terminal vias Itolerance = Difference in the current values of terminal vias. For example, if a difference of up to 5 percent is acceptable, then specify 'Itolerance <= 0.05'. Td = Time duration in micro second or total ‘On Time” period r = Duty ratio
    Power IR/EM supports metal length or width rules above and below vias where the dimensions to be checked for the rules are different for the metal above and the metal below. In this case, Lu/Wu refers to dimensions of the metal above the via and Lb/Wb refers to dimensions of the metal below the via.
    For more information, refer to the Rules for Defining Limit-Based Parameters section in the “Creating the ICT File” chapter in QRC Techgen Reference Manual.
  • Single
    The keyword single can be specified to differentiate between a single square cut via and a square viaarray. The following example shows how to use the single keyword:
    em_jmax_dc_avg PWL 0.022  0.01  jmax_factor 105 1.1  110 1.0 115
    0.9 120 0.8 125 0.7 130 0.6 140 0.5 150 0.4  current_direction up
    Lb > 4 Wb >= 0.05  current_direction up single
  • Current Direction
    current_direction [up | down | both]
    current_direction can be specified for via current direction. The syntax of current_direction is:
    • up - means the current is from down to top and specifies how to use the length and width of the metal below Lb/Wb for the VIA Jmax factor
    • down - means the current is from top to down and specifies how to use the length and width of the metal above Lu/Wu for VIA Jmax factor
    • both - This rule is only applicable when the current direction is uncertain. This rule is unlikely to be applied since the software calculates the current direction
  • Via Range
    via_range <value>
    via_range is an optional parameter which can be used with the via layers. The value of via_range denotes the number of vias for which the calculated limits are valid. via_range is specified as a positive integer.
    Example:
    via Via1 {
       em_model {
          em_jmax_dc_avg 1.11 via_range 1
          em_jmax_dc_avg 2.22 via_range 4
          em_jmax_dc_avg 3.33 via_range 8
       }
       ...
    }
    Current density value of 1.11 mA/via will be applied for viaCount=1
    Current density value of 2.22 mA/via will be applied for viaCount=4
    Current density value of 3.33 mA/via will be applied for viaCount=8
  • The keyword power_rail can be specified to enable the particular rule for power-rail analysis. When power-rail analysis is enabled, a rule with the power_rail keyword will have a higher priority than any other rule.
    em_jmax_dc_avg PWL 0.022 0.01 jmax_factor 105 1.1 110 1.0 115
    0.9 120 0.8 125 0.7 130 0.6 140 0.5 150 0.4 
    Lb > 4 Wb >= 0.05 current_direction up  
    power_rail  Lv2v <= 5 Itolerance <= 0.05
  • The keyword applyR can be used to modify the threshold value of ratio “r’ in AC-Peak analysis. The default value of this parameter is 0.05. Modifying this parameter is useful in scenarios where the value of “r’ is very small. A small “r” value results in an unreasonable increase in the Ipeak_ac limit. You can avoid such scenarios by resetting the value of “r” in the ICT file. For example:
    em_jmax_ac_peak EQ 4.0 r < 0.01 applyR 0.01
    You can specify the threshold ratio by using the applyRThreshold custom variable in the Electromigration Edit/View Custom Parameters form .
    You can also specify the threshold ratio using the environment variable, _vsaApplyRThreshold.
  • In qrcTechFile flow, deltaT can be specified in the GUI using the Edit/View Custom Parameters form. This is used to specify the maximum rise in temperature in degree Celsius, caused due to Joule Heating. It must be a positive value.
    If you do not specify the rise in temperature, Power IR/EM considers a default temperature of 5 degrees Celsius.
    For more information, see the Edit/View Custom Parameters section in the “EM Analysis” chapter.
    Advanced EM rules with power_rail and current_direction keywords require a VPS-XL license.

Reading EM Rules from emDataFile

The following sections detail the EM Rules specification file syntax.

EM Rules Specification File Syntax

  • Single line comments can be added by beginning the line with a semicolon (‘;’). For example:
    ; This is a comment line
  • The Current Density Milliamps Per Via keyword (currentDensityMPV) allows you to define the current density specifications in terms of milliamps per via instead of the standard milliamps per micron. Most foundries specify via and contact current densities in milliamps per via, and the use of this keyword saves you from having to manually convert the foundry specification.
    currentDensityMPV = true

    If the keyword currentDensityMPV is used, then all current density specifications must be stated in terms of milliamps per via.
    When currentDensityMPV is enabled, minW and maxW are defined as the number of vias. For example, 
    (nil layer "myVia" minW 1 maxW 4 currentDensity ((10,85)) 
(nil layer "myVia" minW 4 maxW 8 currentDensity ((20,85))
    Current density=10mA/um if number of vias is >=1 and <4 Current density=20mA/um if number of vias is >=4 and <8
    When currentDensityMPV is disabled, minW and maxW are defined as the total width of a via array. For example,
    Current density=10mA/um if width of via array is >=1 and <4 Current density=20mA/um if width of via array is >=4 and <8
    Width of via array is calculated as (number of vias x minimum width of via)
  • The meshR keyword is set to true when the extraction is done using the meshR option. This option is used to specify regions for mesh resistance extraction. When meshR is specified, the software evaluates the node current direction based on this flag.
    meshR = true
  • deltaT is used to specify the maximum rise in temperature in degree Celsius, caused due to Joule Heating. It must be a positive value.
    If you do not specify the rise in temperature, Power IR/EM considers a default temperature of 5 degrees Celsius. This is an ideal condition where the design must be EM free.
    deltaT can be used in defining the current density limit: Irms
    For example, 
    (nil layer "M1"   minW 0.09 maxW 12.0 currentDensity (((sqrt(18.33*deltaT*(w-.016)*(w-.016)*(w-.016+.352)/(w-.016+.0443) )/w) , 110)))
  • preExecuteFiles
    Takes the list of custom files that are pre-loaded when reading the emDataFile.
    Examples:
preExecuteFiles=(“custom.il”) preExecuteFiles=( getShellEnvVar("myPreExecuteFile"))

where the filename is defined in the environment variable as “myPreExecuteFile”. You can specify any name as the environment filename.

preExecuteFiles=(“customFile1.il” “customFile2.il”)

  • Minimum Current (minI) can be used to limit the reporting of small currents which are of little or no interest during either Power or Signal EM analysis. The value of minI is specified in Amps (A). Any wire segments that have current values less than minI will not be displayed in the EM results windows or reported in the EM result file. The default value of minI is 0, so if it is not specified in the emDataFile it will have no effect on the analysis.
    minI = 1e-9
  • The EM recoveryFactor is used to calculate the effective DC electromigration in bipolar current wave forms. This accounts for some 'sweep back recovery' of EM during the negative portion of the current wave form. The recovery factor is specified in the EM Rules specification file as:
    recoveryFactor = <value>
    where the user-specified value ranges from 0 to 1.0
    The following equation shows how the recovery factor is used by the software while calculating the total average EM current:
    I (avg) = max( I+, I- ) - (recovery factor)* min( I+, I-)
    where I+ is the average of I in the positive direction and I- is the average of I in the negative direction
    The tool errors out if the recoveryFactor value is not equal to 1.0 and when simulation data is in the Power IR/EM-specific format. This is because there is not enough simulation data available in this format for calculating the recoveryFactor. However, recoveryFactor is supported with simulation data in the Power IR/EM-specific format if the simulator version used is Spectre/APS 11.1 ISR19 and above.
  • The recoveryFactorList is used to specify the list of layers and the corresponding recovery factors to be used for these layers during EM analysis. The following example shows how different recovery factors are specified for layers, Via1 and mt1.
    recoveryFactorList = (("Via1" 0.7) ("mt1" 0.3))

    If both recoveryFactor and recoveryFactorList are specified, the list of layers and their corresponding values specified using the recoveryFactorList keyword will get preference.
  • The durationTime is used to specify the maximum time period for which a violation is allowed. For example, if you set the duration time to 1u(sec), then any violations that occur for durations less than 1u are ignored. This helps you eliminate short duration violations that do not have an impact on electromigration. This, in turn, reduces the verification time taken for EM analysis.
    durationTime = <value>
  • The cadGrid specifies the minimum width that Power IR/EM uses when a parasitic resistor (presistor) does not contain width information or has a width that is less than the specified cadGrid. This ensures that no presistor will have a width less than cadGrid. When Power IR/EM changes the width of a presistor to meet the cadGrid a warning message is written to the cds.log. The cadGrid can also be specified in the tech lib. Syntax to specify cadGrid is:
    cadGrid = 0.002
    The unit of measurement is micrometer (um) or micron.
  • The routing layers are the names of the DFII layers that are used for routing. Include only the poly and metals. An example follows:
    routingLayers = ("poly1" "m1" "m2" "m3" "m4" "m5" "m6" "m7" "m8" "m9" "MD")
  • The via layers are the names of the DFII layers. Look for viaLayers in the Virtuoso technology file, for example:
    ( Metal1      Cont            Poly       )  // the via layer is Cont
    An example of the viaLayers syntax follows:
    viaLayers = ( "cw" "v1" "v2" "v3" "v4" "v5" "v6" "v7" "v8")
  • The cross-reference layers (xrefLayers) contain a list of layer names in the following format:
    ( ( "QRCName" ( "currentDensityName" "DFIIName" ) )
    • For conductor layers, QRCName refers to the pro_layer name in the QRC layerSetupFile or p2lvsfile. For via layers, QRCName refers to the ext_cont layer names. For example:
      === layerSetupFile ===
      pro_layer=myM1 ext_layer=m1_par, m1
      pro_layer=myM2 ext_layer=m2_par,m2 model=M2mod
      pro_cont=myV1 ext_cont=v1_par, v1
      pro_cont=myV2 ext_cont=v2_par, v2 model=V2mod
      === emDataFile ===
      xrefLayers=(
      ( myM1 (M1 M1))
      ( M2mod (M2 M2))
      ( v1_par (V1 V1))
      ( v1 (V1 V1))
      ( V2mod (V2 V2))
    • currentDensityName is the layer name used in the current density specification declared by the foundry.
    • DFIIName is the layer name in the DFII technology directory.

    The above three layer names might all be same or different depending on how the rules files are written. For example:
    xrefLayers = (
( "poly1con" ("cw" "cw"))
( "poly" ("poly1" "poly1")))

    In general, QRCName should match the parasitic resistor model names that are found in the extview.tmp file in the QRC run directory. However, if the layer name matching fails for the specified QRCName, the matching will be done using currentDensityName and DFIIName.
    If the same via layer has different resistance values, for example, diffusion contact and poly contact, then they should be mapped under different currentDensityNames so that the software can use the appropriate resistance values. For example:
    xrefLayers(
(diffCont (CONTdiff CONT))
(pcCont (CONTpc CONT))
)
    (nil layer "CONTdiff" minW 0.0 maxW -1.0 minL 0.0 maxL 1.0 res 2.0 currentDensity ((1.0, 27))
    (nil layer "CONTpc" minW 0.0 maxW -1.0 minL 0.0 maxL 1.0 res 3.0 currentDensity ((1.0, 27))
    OR
    (nil layer "CONTdiff" minW 0.0 maxW -1.0 minL 0.0 maxL 1.0 currentDensity
    ((1.0, 27))
    (nil layer "CONTpc" minW 0.0 maxW -1.0 minL 0.0 maxL 1.0 currentDensity ((1.0, 27))

    If "res" is not explicitly specified, it will be retrieved from the QRC run directory which is assumed to be within the specified LVS run directory.
  • CurrentDensSpecList refers to the current density specification declared by the foundry. The syntax allows one layer to support two different current density specifications according to the width of the material.
    There are seven Current Density Specifications in the emDataFile: avgCurrentDensSpecList, avgAbsCurrentDensSpecList, rmsCurrentDensSpecList, DCrmsCurrentDensSpecList, peakCurrentDensSpecList, ACpeakCurrentDensSpecList, and userDefinedCurrentDensSpecList.
    The DC operating point (DCOP) analysis will use the avgCurrentDensSpecList. ACpeakCurrentDensSpecList is used when you select AC-Peak analysis on the EM form. See EM Analysis of the Power Grid, in the “EM Analysis” chapter. Each specification in the emDataFile can be unique with its own currentDensity and temperature.
    An example of a current density specification for a given layer is as follows:
    (nil layer "v1" minW 0.0 maxW -1 res 0.68 currentDensity ((1.2100,100) (1.2110,110) (1.2125,125)))
    • nil is a reserved keyword indicating the start of the line.
    • layer “poly1” is the layer keyword followed by the specific layer name to apply the current density line. The layer name is the current density layer name from the layer cross-reference section of the emDataFile.
    • minW is the minimum line width in microns.
    • maxW is the maximum line width. When set to -1.0, it represents infinity.
    • res is the resistance in ohms.
      • For contacts and via layers res should be set to the total resistance of a minimum width contact.
    • currentDensity is specified in milliamps per micron at a specified junction temperature. This can be represented in multiple (currentDensity, temperature) pairs.
      currentDensity ((1.2100,100) (1.2110,110) (1.2125,125))
      • Most foundries specify via and contact current densities in milliamps per contact or milliamps per square micron. The foundry specification must be converted to milliamps per micron. For example, if the foundry specification provides a current density of 0.4mA/via and the via is 0.2micron X 0.2micron, then the currentDensity value is 0.4 / 0.2 = 2mA/um
        With the introduction of the currentDensityMPV keyword, current densities can now be specified in milliamps per contact as provided by the foundry so no manual conversion is required.
    • temperature is specified in degree Celsius.
      The software compares the simulation temperature with this temperature. When the simulation temperature is not in the temperature range specified above, the software behaves in the following manner:
      • If the simulation temperature is less than the specified minimum temperature, the current density value is picked for the specified minimum temperature and not for the simulation temperature.
      • If the simulation temperature is greater than the specified maximum temperature, the current density value is picked for the specified maximum temperature and not for the simulation temperature.

      When the simulation temperature falls within the specified temperature range, the software interpolates the value using a formula that is provided in the example below.
      The tool uses temperature in Kelvin while doing the calculation using the interpolation equation.
      Examples:
      In these examples, T is the simulation temperature, T1 is the specified minimum temperature, and T2 is the specified maximum temperature.
      currentDensity (( RT1, T1) (RT2, T2)) where T1 < T2
      If T < T1; current density will be taken for temperature T1
      If T > T2; current density will be taken for temperature T2
      If T1 < T < T2; The tool will interpolate for the temperature derating factor that is derived from the formula: TTF=A*J^(-n)*exp(Ea/kT)
      RT = RT1*exp((Ea/kT)*(1/T-1/T1))
      where Ea/kT=(T1*T2/(T1-T2))*ln(RT2/RT1)
      You can enable linear interpolation for RT factor at temperature T by setting the _vsaLinearInterpolation variable to true.
    The following is an example of the peakCurrentDensSpecList parameter, where the currentDensity is dependent on the width of the presistor:
    peakCurrentDensSpecList = (
    (nil layer "m1" minW 0.0 maxW 0.16 currentDensity ( (1.434*3.9*w,100) (3.9,110)
    (0.387*3.9,125)))
    (nil layer "m1" minW 0.16 maxW -1.0 currentDensity ( (14*w*deltaT,100) (7.0-
    0.16,110) (5.3,125)))
    In the preceding example, the currentDensity for layer “m1” will be calculated by the equation using the actual line width depending upon whether it is greater than the 0.16 maxW specified as the breakpoint between the two lines.
    The following is an example of the peakCurrentDensSpecList parameter, where the currentDensity is dependent on length of the presistor:
    (nil layer "Metal1" minW 0.0 maxW -1.0 minL 0.0 maxL 1.0 
    currentDensity ((1.0, 27) (1.01,110) (1.02,125)) )  
    (nil layer "Metal1" minW 0.0 maxW -1.0 minL 1.0 maxL 2.0
    currentDensity ((3.1*L*deltaT, 27) (4.11*L*deltaT,110) (4.32,125)) )  
    (nil layer "Metal1" minW 0.0 maxW -1.0 minL 2.0 maxL -1.0
    currentDensity ((1.2, 27) (1.21,110) (1.22,125)) )
    The following is an example of the peakCurrentDensSpecList parameter, where the currentDensity is dependent on the length of the routing layers connecting to the via:
    In the following example, the minW and maxW values are for Via layers while minL and maxL values are for the metal layers adjacent to the via layers:
    (nil layer "Via1" minW 0.0 maxW -1.0 minL 0.0 maxL 1.0 
    res 2.0 currentDensity ((1.0, 27) (1.01,110) (1.02,125)) )  
    (nil layer "Via1" minW 0.0 maxW -1.0 minL 1.0 maxL 2.0
    res 2.0 currentDensity ((3.1*L*deltaT, 27) (4.11*L*deltaT,110) (4.32,125)) )  
    (nil layer "Via1" minW 0.0 maxW -1.0 minL 2.0 maxL -1.0
    res 2.0 currentDensity ((1.2, 27) (1.21,110) (1.22,125)) )  
    )
    The minL and maxL specified in the emDataFile refer to the effective path length and not to the individual presistor length. The Jmax will be picked based on the path length of presistor and temperature. In case the Jmax equation contains the L variable it will be replaced with the path length.
    Effective path length is the longest path length in a segment. If there is a loop in the segment, the total length of the segment is taken as the effective path length.

The current density rules can be specified in terms of both width and length. For example the current limits for layer M1 can be described as below:

Metal length in um Jmax

L >= 20um 1.509 x (w-0.016)

20 > L > 5 (20/L) x 1.509 x (w-0.016)

L <= 5 4 x 1.509 x (w-0.016)

In the above case, the “w” will be replaced with the presistor width and “L” will be replaced with the path length of the metal route of which the presistor is a part.

  • Use the technologyNode = 40 parameter to specify the technology-dependent calculation. If the technologyNode >= 40 then the effective length is calculated by the earlier method in which all the lengths are added.
    You can also specify this using the environment variable, _vsaTechnologyNode 40. If both are specified, the technologyNode parameter specified in the emDataFile will have a higher priority than the corresponding environment variable.
  • In case of via, you can specify upper metal layers and bottom metal layers in terms of both length and width. For example:
    Metal length/width in um jmax
    Lu < 5.0um                            1.2
    Lb < 7.0um                             1.2
    Wu < 4.0um                             1.2
    Wb < 6.0um                            1.2
    where:
    Lu = Upper metal length
    Lb = Bottom metal length
    Wu = Upper metal width
    Wb = lower metal width
    For example:
    (nil layer "Via1" minW 0.0 maxW -1.0 minL 2.0 maxL -1.0 
    minLu 0.0 maxLu 5.0 minLb 0.0 maxLb 7.0 minWu 0.0 maxWu 4.0 minWb 0.0 maxWb 6.0
    res 2.0 currentDensity ((1.2, 27) (1.21,110) (1.22,125)) )

    The (min/max) and (Lu/Lb/Wu/Wb) are optional parameters. If not specified, than min* is taken as 0.0 and max* is -1.0.
  • The layerstack contains a list of layer names and corresponding upper metal and bottom metal layer names.
    ( ( "LayerName" ( "upperMetalLayerName" "bottomMetalLayerName" ) )
    • LayerName is the layer name specified in the extracted view
    • upperMetalLayerName is the upper metal layer name in the extracted view
    • bottomMetalLayerName is the bottom metal layer name in the extracted view
  • The Ignore Missing Spec (ignoreMissingSpec) flag helps you to ignore layers for which current densities are missing.
    ignoreMissingSpec= true
    The default value of the flag is false.
    If ignoreMissingSpec is not specified, the tool selects the default current density as 0.9 for missing current densities.
  • The Via Width list (viaWidthList) specifies the minimum via width. In the following example:
    viaWidthList = ( ( "Cont" 0.2 ) ( "Via1" 0.2 ) )
    • Cont is the DFII name of the via layer.
    • 0.2 is the minimum width of the default via for this layer.
      If the via width information is not found in the DFII technology file, it must be included in the emDataFile. In this case, it must include all of the vias specified in the viaLayers section of the emDataFile or an error will occur.
  • The Via Width list (extViaWidthList) specifies the minimum via width. In the following example:
    extViaWidthList = ( ( "extCont" 0.2 ) ( "extVia1" 0.2 ) )
    • extCont is the QRC (Extraction) name of the via layer.
    • 0.2 is the minimum width of the default via for this layer.
      viaWidthList will be ignored if extViaWidthList is specified.
  • The Via Length list (viaLengthList) specifies the via length for use in calculating the area of vias that are not square. This feature is only required when currentDensityMPV is set to “true”. If the vias in the design are square, then you can specify viaWidthList/extViaWidthList without viaLengthList or extViaLengthList.
    viaLengthList = (Via1 0.12)
    • Via1 is the DFII name of the via layer.
  • The Via Length List (extViaLengthList) specifies the via length for use in calculating the area of vias that are not square. This feature is only required when currentDensityMPV is set to "true". If the vias in the design are square, then you can specify viaWidthList/extViaWidthList without viaLengthList or extViaLengthList.
    extViaLengthList = (extVia1 0.12)
    • extVia1 is the QRC (Extraction) name of the via layer.
      viaLengthList will be ignored if extViaLengthList is specified.
  • The dfiiUnits variable specifies the units of the via length to the tool, if the tool is unable to identify the length units by itself. Valid values for this variable are “micron” and “meter”.
    dfiiUnits = “micron” or dfiiUnits = “meter”
  • The following filtering options allow you to filter resistors based on either resistor names or layer names:
    • selectLayers
      Selects resistors based on the extracted layer name. All resistors on other layers will be ignored for the analysis.
      Example:
      selectLayers  =  (“Via1” “mt1” “mt2”)
      selectLayers  =  (“Via1” “m.*”)
      SKILL regular expressions are supported.
      You can also provide the list of selected layers using the _vsaSelectLayers environment variable.
    • selectResistors
      Selects resistors by their names. All other resistors will be ignored for the analysis.
      Example:
      selectResistors  =  (“rh12” “re45” “rf25”)
      selectResistors  =  (“rh.*” “re.*”)
      SKILL regular expressions are supported.
      You can also provide the list of selected resistors using the _vsaSelectResistors environment variable.
    • ignoreLayers
      Ignores resistors based on the extracted layer name. All resistors on other layer will be selected for the analysis.
      Example:
      ignoreLayers  =  (“Via1” “mt1” “mt2”)
      ignoreLayers  =  (“Via1” “m.*”)
      SKILL regular expressions are supported.
      You can also provide the list of ignored layers using the _vsaIgnoreLayers environment variable.
    • ignoreResistors
      Ignore resistors by their name. All other resistors will be selected for the analysis.
      Example:
      selectResistors  =  (“rh12” “re45” “rf25”)
      selectResistors  =  (“rh.*” “re.*”)
      SKILL regular expressions are supported.
      You can also provide the list of ignored resistors using the _vsaIgnoreLayers environment variable.
      The parameters specified in emDataFile have higher priority than their corresponding environment variables. If both are specified, the parameters specified in the emDataFile will be selected over the corresponding environment variable values.
      Also, the selection list has a higher priority than the ignore list. If selectLayers or selectResistors parameters are already provided, then the values specified using ignoreLayers and ignoreResistors parameters will be ignored.
    • DesignResistors
      Specifies the list of design resistor cell names. For each cell name, you can specify the resistor property name, width property name, length property name, and units for width and length property names for which the design resistors are to be analyzed. Specifying the resistor property name, width property name, length property name, and unit name is optional. If any of these property names is not specified, the software uses “r”, “w”, l”, and “meter”, respectively, as the default values for these properties.
      The width and length property names are specified when the names of these properties are different from the “w” and “l” in the extracted view.
      Syntax:
      DesignResistors  =  (“CellName1” “CellName2” “CellName3”)
      DesignResistors  =  (“CellName.*”) 

      SKILL regular expressions are supported.
      Example:
      DesignResistors = (
                      ("nplusres"         "r" ("wSim" "meter") ("lSim" "meter"))
                      ("pplusres"        "r" "wSim" "lSim")
                      ("qplusres"         )
      )
      Where: nplusres, pplusres, and qplusres refer to the design resistor cell names r = resistor property name wSim = width property name meter = unit for width lSim = length property name meter = unit for length
      The default unit for both the width and length property name is meter. However, the software also accepts “micron”, “millimeter”, “centimeter”, “mill”, and “inch” as units for these property names.
      For details, see Analyzing EM Across Design Resistors in the “EM Analysis” chapter.

User Procedures for Electromigration JMAX Checking

Power IR/EM has its own methodology to perform average, peak, and RMS analyses. But it also provides you options to define your own methodology to calculate current densities.

In EM flow, after you calculate the current density of a presistor, you can compare it with the current density limits to find violations. The current density limits are read from the emDataFile that you provide. In addition to providing the emDataFile, you can also specify that the current density limits be calculated dynamically.

Power IR/EM provides the following procedures to the users for rms, DCrms, avg, avgAbs, peak, peakAC, and custom analysis respectively:

vsaGetRmsJmax
vsaGetDCRmsJmax
vsaGetAvgJmax
vsaGetAvgAbsJmax
vsaGetPeakJmax
vsaGetPeakACJmax
vsaGetCustomJmax

For example, if you think that peak current density limits cannot be specified using the emDataFile, you can write a SKILL function vsaGetPeakJmax() in a .il file and load that before starting the EM analysis. The parameters passed to this function are fixed and the function returns a float value for Jmax. Also, this function will be called for each presistor so that you can specify different values for Jmax.

An example of a SKILL function is shown below:

procedure(vsaGetPeakJmax(@key
(r 1.0)
(w 1.0)
(temp 25)
(layer "METAL1")
(pResistorName lowerCase(resName) 
(Hierarchy hierarchy)
(parameters ’nil)
; 1.0, 1.0, 25, METAL1, lowerCase(resName), and hierarchy are default values used only for SKILL syntax
