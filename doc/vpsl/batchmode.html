Overview

Power IR/EM can be run as a batch command, running from Virtuoso in no-graph mode. Batch mode can be used to perform IR Drop analysis, EM analysis, and for power-grid view (PGV) creation.

Batch Command Syntax

The batch mode is executed from the command line as follows:

vsabatch -cmd <cmd_file> -log <log-file>

Where:

<cmd_file> is an ASCII file of SKILL commands to define the input data, control the execution of analysis, and specify the output location. For information about the various SKILL commands, see Command File Syntax. To view an example, cmd_file, see Sample Command File.

Starting a line with a semicolon (‘;’) indicates a comment.

<log_file> specifies the name and location of a log file of the execution. If -log is not specified, the default log file cds-vsa.log will be used.

In batch mode, environment variable settings are not used. Therefore, values of environment variables must be specified as commands in the command file. These are _vsa_em_data_file, _vsa_simulation_directory, and _vsa_qrc_run_directory.

_vsa_extracted_lib_cell_view “library” “cell” “view”

_vsa_extracted_lib_cell_view "library" "cell" "view"

Specifies the RCX extracted view used as input for the analysis. This is a required command.

Parameters

library

Specifies the name of the library.

cell

Specifies the name of the cell within the library.

view

Specifies the name of the view within the cell.

Example

The following command specifies the RCX extracted view used as input for the analysis:

_vsa_extracted_lib_cell_view "amsPLL" "vco" "av_R"

_vsa_testbench_lib_cell_view “library” “cell” “view”

_vsa_testbench_lib_cell_view "library" "cell" "view"

Specifies the schematic testbench. This is an optional command. However, if a schematic testbench is not specified then the _vsa_hierarchy command must be specified.

Parameters

library

Specifies the name of the library.

cell

Specifies the name of the cell within the library.

view

Specifies the name of the view within the cell.

Example

The following command specifies the schematic testbench used as input for the analysis:

_vsa_testbench_lib_cell_view "amsPLL" "tb_vco_bak" "schematic"

_vsa_hierarchy “hierarchy_name”

_vsa_hierarchy "hierarchy_name"

Specifies the hierarchy of the design. For a single level of hierarchy, this is an optional command. If the hierarchy is specified, the _vsa_testbench_lib_cell_view command must also be specified.

For multiple levels of hierarchy, if you want to run in batch mode, then the testbench lib/cell/view name must be provided in addition to the hierarchy name. In this case, you do not need to specify the pinmap.

Parameter

hierarchy_name

Specifies the schematic cell name.

Example

The following command specifies the hierarchy of the design:

_vsa_hierarchy "/X1"

_vsa_pinmap "extrPinName" "testBenchPinName"

_vsa_pinmap "extrPinName" "testBenchPinName"

Specifies the pin name mapping between the testbench, through which the simulation was completed, and the specified extracted view hierarchy. This command is required when _vsa_hierarchy is used for a single level of hierarchy. It is not required when the _vsa_testbench_lib_cell_view command is used.

Parameters

extrPinName

Specifies the extracted view pin name.

testBenchPinName

Specifies the testbench or schematic pin name.

Example

The following commands specify the pin name mapping between the testbench and the extracted view.

_vsa_pinmap  "p0" "/out0" 
_vsa_pinmap  "p135" "/out3"

_vsa_simulation_directory "sim_dir_path"

Specifies the input simulation run. This is a required command.

sim_dir_path

_vsa_pres_cellname “cellname”

_vsa_pres_cellname "cellname"

Specifies the cell name of the parasitic resistor used by RCX, and added to the extracted view. This command is optional.

Default: “presistor”

Parameter

cellname

Specifies the cell name of the parasitic resistor within the cell.

Example

The following command specifies the name of the parasitic resistor used by RCX:

_vsa_analysis_data “TRANSIENT” | “DCOP”

_vsa_analysis_data "TRANSIENT" | "DCOP"

Specifies the type of analysis to perform. This command is optional.

For transient analysis, separate start and stop times can be specified for both IR Drop and EM analysis using the _vsa_analyze_ir and _vsa_analyze_em commands. If start and stop times are not specified, the default is the full duration of the simulation.

Default: “TRANSIENT”

Parameters

TRANSIENT

Specifies that transient analysis should be performed.

DCOP

Specifies that DC operating point analysis should be performed.

Examples

  • The following command specifies that Transient analysis be performed for the full duration:
    _vsa_analysis_data "TRANSIENT"
  • The following command specifies that DCOP analysis be performed:
    _vsa_analysis_data "DCOP"

_vsa_vavo_db_enable t | nil

_vsa_vavo_db_enable t | nil

Specifies whether vavo.db database will be used for IR drop/EM analysis or not. When set to nil, PSF database is used. This command is specified only when you have vavo.db database in your simulation directory.

Default: t

Parameters

t

Specifies that vavo.db database will be used for analysis.

nil

Specifies that PSF database will be used for analysis.

Example

The following command specifies that the analysis should be performed using the vavo.db database:

_vsa_vavo_db_enable t

exit

exit

Closes the virtuoso -nodisplay application and is used at the end of the command file. If exit is not specified, the program will not terminate after the IR/EM analysis is complete. It will wait for further input from you.

This is not desirable in an application running without display.

Example

The following command is used at the end of the command file:

exit

_vsa_ir_pin_name “all” | “pin1 pin2 ...”

_vsa_ir_pin_name "all" | "pin1 pin2 ..."

Specifies the pin name(s) for which IR Drop analysis is to be performed. These pin names should come from the layout name space. Pin names can be specified either with this command or with the _vsa_ir_pin_name_file command.

_vsa_em_job_name “jobName” 

_vsa_em_job_name "jobName"

Prepends a job name to output file names.

Parameter

jobname

Specifies a job name provided by you for the current run.

During the Power IR/EM execution, the following default files are created internally. These files are removed when the job is completed.

vsa2CmdFile.cmd
vsa2Results.txt
vsa2Input.txt

To run multiple jobs simultaneously, use the _vsa_em_job_name command so that the output files for different runs are prepended with different job names. For example: 

_vsa_em_job_name "myjob3"

The following output files are created. These are removed when the job is completed. 

myjob3_vsa2CmdFile.cmd
myjob3_vsa2Results.txt
myjob3_vsa2Input.txt

The following file is output for viewing:

myjob3.EM.results
