Load a Predefined Set of Libraries on Cadence Help Startup

Libraries from other locations can be configured to make them visible to those who are accessing Cadence Help from a particular location. For this, you need to create a help.ini file at the location:

<inst_dir>/tools.<port>/cdnshelp/config/

The content of the help.ini file will be:

[library]

<Path of Library 1>

<Path of Library 2> (For example: /servers/etstore2/et/setup/111/ET111101.2012-01-21/lnx86/doc/xmlreg/lnx86/library.lbr)

Example:

[library]

/home/usr1/cadence/IC615/doc/xmlreg/library.lbr

/servers/etstore2/et/setup/111/ET111101.2012-01-21/lnx86/doc/xmlreg/lnx86/library.lbr

If you have a network installation of Cadence Help, you might have more than one user accessing Cadence Help. If multiple users need to access different libraries, you can set a CDS_SITE environment variable that points to a common help.ini file. You can specify multiple libraries in this file. The CDS_SITE environment variable ensures that all the libraries in the help.ini file can be accessed by more than one user accessing Cadence Help installed on a network location.


Using Environment Variables

You can also specify the environment variables and the path of the library in the help.ini file. This enables site administrators to configure Cadence Help installation across the site users.

AVS_INSTALL

If you have your Cadence tools installed at /grid/avs/install/incisiv/latest, you set an environment variable, AVS_INSTALL, which points to the Cadence tools installation directory (using the command setenv AVS_INSTALL /grid/avs/install/incisiv/latest).

Now, you can specify the environment variable in the help.ini file so that whenever you change the variable to point to a new installation, the corresponding documentation library is loaded. In the example above, the help.ini file looks as shown here:

[library]

$AVS_INSTALL/doc/xmlreg/library.lbr

CDNSHELP_DOC

The documentation hierarchies specified in the CDNSHELP_DOC variable are also loaded by default. For example, if you specify:

setenv CDNSHELP_DOC  <install_dir1>/doc:<install_dir2>/doc

Cadence Help loads the documentation hierarchies for install_dir1 as well as install_dir2.

The above command works for Linux platform. Windows platform does not support ':' as a delimiter. Use the following syntax for setting variable:

set CDNSHELP_DOC  <install_dir1>/doc;<install_dir2>/doc

You can use a combination of command-line switches with the environment variables to launch Cadence Help. For example, you can use the switches such as -hierarchy and -openpage  to launch and load a topic directly into Cadence Help.

For example, 

cdnshelp -hierarchy $MYLIB/doc/xmlreg/library.lbr -openpage mydoc:mytopic

In this example, the mytopic page of the mydocbook loads into Cadence Help.


  • You can run Cadence Help in Offline mode, without making any network calls. Specify the following command:
 setenv CDNSHELP_OFFLINE t

USE_PATH

If your Cadence Help installation location is present in the PATH variable, you can specify the USE_PATH entry in the help.ini file. This enables Cadence Help to load all the documentation libraries present in the PATH. A sample help.ini file looks as shown here:

[library]

$AVS_INSTALL/11.1/11.10.076/doc/xmlreg/library.lbr

USE_PATH

CDNSHELP_ENABLE_LIBS

If you want Cadence Help to enable all the available libraries on startup, add the CDNSHELP_ENABLE_LIBS variable in the help.ini file.

Disabling Cadence Help Features

  • To disable Show Usage Statistics, add the following entry in help.ini: 
REMOVE_STATS
  • To disable Glossary, add the following entry in help.ini: 
REMOVE_GLOSSARY

  • To limit network calls, sharing information like context, release version, call mode etc., while launching Cadence Help, add the following entry in help.ini: 

REMOVE_TRACK 

  • To launch Cadence Help in offline mode, completely cutting the network calls, set the following environment variable:

 setenv CDNSHELP_OFFLINE t
